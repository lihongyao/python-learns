# 模块

## 定义

含义：在Python中，一个 `.py` 文件即被视为一个模块，其中可以定义函数、变量等。

分类：

1. 内置模块：Python 内置的模块，可以直接导入使用。
2. 第三方模块：使用 `pip install module_name` 指令安装的模块。
3. 自定义模块：自己在项目中定义的一些模块。

## 分类

### 内置模块

Python提供了一个强大的标准库，内置了许多非常有用的模块，如 `copy`、`random`、`time`、`os` 等，这些模块可以直接使用，为开发者提供了极大的便利。

示例：

```python
# 导入 random 模块
import random

num = random.randint(1, 3)
print(num)
```

### 第三方模块

是由Python社区或其他开发者创建的，不是Python官方标准库的一部分。这些模块提供了各种额外的功能和工具可以帮助开发者更高效地完成编程任务。

在使用第三方模块之前，需要先下载和安装它们。Python提供了一个名为 `pip` 的包管理工具，用于下载和安装第三方模块。

在命令行或终端中，使用 `pip` 安装第三方模块的命令通常如下:

```shell
$ pip install 模块名
```

示例：

```shell
$ pip install requests
```

### 自定义模块

自定义模块是指自己在项目中定义的一些模块。在创建自定义模块时，应确保命名遵循 Pvthon 的标识符规定和变量的命名规范，并且要避免与 Python的 内置模块以及已安装的第三方模块命名发生冲突，以防止模块功能出现异常或无法使用。

如 abc.py 文件与 Python 的内置模块 abc 发生冲突，该文件内的功能无法使用。

## 导入模块

随着程序的不断扩展和完善，一个.py 文件中可能会包含大量的代码行、函数和变量等。为了便于维护和提高代码的可重用性，我们通常会将代码拆分为多个文件，即多个模块。这样做的好处是，当我们需要在其他模块中使用某个模块的功能时，只需简单地导入该模块即可，从而提高了代码的可维护性和重用性。

> **注意**：
>
> 1. 导入模块语句一般建议写在 ·py 文件的开头部分，这样做有助于代码的组织和阅读，同时也符合一般的编程习惯。
> 2. 同一个模块在Python程序中只需导入一次，因为首次导入时模块就已经被加载到内存中，无需再次导入，这样既节省了资源，也避免了潜在的命名冲突，有助于提高程序的效率和可维护性。

### `import 模块名 (as 别名)` ：导入模块所有内容

新建 `test.py`  文件，内容如下：

```python
import random

version = "v1.0.0"


def sum(a: int, b: int):
    return a + b


def get_random_num():
    return random.randint(1, 100)

```

需求1：导入 `test.py` 模块，并调用其内部的 `get_random_num` 函数

```python
import test

# 在需要使用模块中的变量，函数或者类时，需要在前面加上 “模块名.” 前缀，以便正确地引用他们
# 使用功能：模块名.功能名
print(test.get_random_num())
```

注意：在导入模块时，如果模块名较长不易记忆，可以使用 as 关键字为其设置一个别名。之后，便可以通过这个别名来方便地调用模块中的变量、函数或类等。

```python
# 导入名为 test 的模块，并为其设置别名 t
import test as t

print(t.version)
print(t.sum(1, 2))
```

在使用 `import` 语句导入模块时，会导入模块中的所有内容。然而，如果只想导入模块中的部分内容，可以使用 `from ... import ...` 语句。使用这种方式导入后，无需再添加 **`模块名.`** 作为前缀，直接通过具体的变量名、函数名或类名等即可访问导入的内容。

### `import 模块名 import 功能名`：从模块中导入置顶功能

需求 2：从 `test` 模块中导入 `sum` 函数，并执行该函数。

代码实现：

```python
from test import sum
print(sum(1, 2))
```

`from ... import ...`语句可以实现导入模块的所有内容，通过使用 `from ... import *` 语法来实现。

```python
from test import *

print(version)
print(sum(1, 2))
print(get_random_num())

"""
v1.0.0
3
73
"""
```

> **注意**：在使用 `from ...import ...` 语句导入模块内容时，必须特别留意，确保不同模块间的内容不可重名，否则会产生冲突。具体来说，若不同模块中存在同名的变量、函数或类，后导入的内容会覆盖先导入的。为了避免这种情况，推荐使用 `import` 语句进行导入，这样可以通过模块名来明确区分和使用不同的内容。

# 以主程序的形式执行

每个 Python 模块都内置了一个特殊变量 `__name__`，它用于记录模块的名称，程序通过检查这个变量，可以确定代码是在哪个模块中执行的。如果一个模块是直接被运行的，而不是被导入到其他程序中，那么 `__name__` 变量的值会被设置为 `__main__`。相反，如果这个模块是被导入到其他模块中执行的，那么 `__name__` 变量的值就会被设置为该模块的模块名。

代码示例：

```python
import random

version = "v1.0.0"
def sum(a: int, b: int):
    return a + b
def get_random_num():
    return random.randint(1, 100)

# 测试代码
if __name__ == "main":
    print(__name__)
    print(f"当前版本 {version}")
```

# 包

在Python中，为了解决模块名重复的问题，引入了包(Package)的概念。包是一个分层次的目录结构，用于将一组功能相近的模块组织在一起，这样不仅有助于规范代码结构，还能有效避免模块名之间的冲突。简单来说，包就是一个特殊的"**文件夹**"，用于组织和管理相关的模块。

两种实现方式：

1. `import 包名`
2. `from ... import ... 导入包内指定模块`

## 包的创建和使用

### 创建包

在根目录（可选）中，创建 `utils` 包，如下所示：

```
python-project
.
├── main.py						 # 主程序
├── utils              # 包名（文件名）          
│   ├── __init__.py    # 在使用 Pycharm 创建包时自动生成，用于标识当前文件夹是一个包，引入时会调用
│   ├── a.py           # 包模块 a
│   └── b.py 					 # 包模块 b
```

创建包的目的不是为了直接运行，而是为了被导入并使用。实际上，包可以被视为模块的一种特殊形式，它的本质在于组织和封装多个模块，以便更方便地进行模块化管理和使用。

> `__init__.py`

```python
print("utils/__init__.py is running...")
from . import a  # 显式导入子模块a
from . import b  # 显式导入子模块b
```

> `a.py`

```python
name = "模块a"
```

> `b.py`

```python
name = "模块b"
```

### 使用包（导包）

#### `import 报名(.模块名)`：导包

需求：导入名为 `utils` 的包

代码实现：

```python
import utils
```

注意：在 Python 中，每当包被导入时，`__init__.py` 文件会自动执行，作为包初始化的一个环节。然而，为了保持包的清晰和高效，建议在 `__init__.py` 文件中避免编写过多或复杂的代码。应尽可能保持其内容简洁，主要用于设置包的基本属性、导入必要的模块或子包，以及执行一些轻量级的初始化操作。

需求：导入 `utils` 包下名为 `a` 的模块，并输出 `a` 模块中的 `name` 变量的值

```python
import utils

# 使用功能：包名.模块名.功能名
print(utils.a.name)
```

#### `from ... import ... 导入包内指定模块`

```python
from utils import a
print(a.name)
```

> **注意**：如果 `__init__.py` 文件中没有定义 `__all__` 变量或显式地导入包内的任何模块或子包，那么执行 `from 包名 import *` 时，它只会导入 `__init__.py` 文件中直接定义的内容(如函数、类、变量等)，而不会导入包内的其他模块或子包。

## `__all__` 变量

是一个列表，用于模块和包中控制 `from ... import *` 的导入行为。在模块中定义 `__all__` 列表时，使用 `from 模块名 import *` 将仅导入该列表中指定的元素(如函数、类、变量等)。同样地，当在包下的 `__init__.py` 文件中定义 `__all__` 列表时，执行 `from 包名 import *` 将仅导入该列表中明确指定的模块(如果列表中包含模块名)或 `__init__.py` 文件中直接定义的其他名称(如函数、类等)。
