# 类型转换

不同类型的数据在编程中具有各自独特的特点和用途，这些特点决定了它们在特定情境下的适用性和效率。因此，类型转换成为了一个必要的工具，用于在不同类型之间转换数据，以便优化程序的性能、提升代码的可读性和可维护性，或者仅仅是为了满足特定的编程需求。

在Python中，类型转换是将一种数据类型的值转换为另一种数据类型的过程。Python提供了几种内置函数来支持数据类型的转换。

## 数值类型转换

- `int(x)`：将 `x` 转换为整数（截断小数部分）
- `int(x, base)`：将 `x` 从指定进制 `base` 转换为整数
- `float(x)`：将 `x` 转换为浮点数
- `complex(real, imag)`：创建复数（`real` 为实部，`imag ` 为虚部）
- `complex(x)`：将字符串或数字转换为复数

## 布尔类型转换

- `bool(x)`：将 `x` 转换为布尔值（`False` 对应空值/零，其余为 `True`）

## 字符串转换

- `str(x)`：将 `x` 转换为字符串
- `repr(x)`：将 `x` 转换为表达式字符串（可被 `eval()` 解析）
- `eval(str)`：执行字符串中的Python表达式并返回结果

## 序列类型转换：

- `list(s)`：将序列 `s`（如元组、字符串、集合）转换为列表
- `tuple(s)`：将序列 `s` 转换为元组
- `set(s)`：将序列 `s` 转换为可变集合
- `frozenset(s)`：将序列 `s` 转换为不可变集合

## 字典转换

- `dict(d)`：将键值对序列 `d`（如元组列表）转换为字典

## 字符串与编码转换

- `chr(x)`：将整数`x`转换为Unicode字符
- `ord(x)`：将字符`x`转换为ASCII/Unicode整数值

## 进制转换

- `bin(x)`：将整数 `x` 转换为二进制字符串
- `oct(x)`：将整数 `x` 转换为八进制字符串
- `hex(x)`：将整数 `x` 转换为十六进制字符串

> **提示**：若转换失败（如非数字字符串转整数），会抛出 `ValueError` 异常

# 深浅拷贝

在 Python中，当你将一个可变对象赋值给一个新的变量时，实际上是在创建这个对象的一个新引用（而不是对象的副本）。这意味着两个变量现在都指向内存中的同一个对象。因此，如果你通过任何一个变量对这个对象进行修改，这些修改将会影响到对象本身，进而影响到所有指向该对象的变量。

```python
dict1 = {"name": "张三", "age": 18}
dict2 = dict1
print(dict2)  # {'name': '张三', 'age': 18}

dict2["name"] = "李四"
print(dict1)  # {'name': '李四', 'age': 18}
print(dict2)  # {'name': '李四', 'age': 18}

# 比较两个变量所引用的内存地址是否相同：如果内存地址相同，则表明这两个变量引用的是同一个对象，否则表明引用的是不同的对象
print(id(dict1) == id(dict2))  # True
```

总结：

在Python中，当使用赋值语句(如 `a =b`)将一个可变对象(如列表、字典等)赋值给另一个变量时，这两个变量实际上会完全共享对同一个对象的引用。这意味着，通过任何一个变量对该对象所做的修改(如添加、删除元素或更改值)都会直接影响到原始对象，并因此被另一个变量所共享，导致两个变量所观察到的对象状态是一致的。

然而，在某些情况下，我们需要保留一份数据的原始内容不变，以便在不影响原始数据的前提下对数据进行处理或修改。此时，仅仅使用赋值操作(=)是不明智的，因为它会导致新的变量与原始变量共享同一个对象的引用。为了解决这个问题，Python提供了copy 模块，该模块提供了两种主要的拷贝方法：

- 浅拷贝：`copy()`
- 深拷贝：`deepcopy()`

使用方法：

```python
# 导入模块
import copy

# 定义列表
li1 = [1, 2, [3, 4]]
# 深拷贝
li2 = copy.deepcopy(li1)
# 浅拷贝
li3 = copy.copy(li1)
```

## 浅拷贝

在Python中，浅拷贝会创建一个新的对象，但这个新对象只拷贝了原始对象的第一层数据。如果原始对象包含嵌套的可变对象(如列表、字典等)，浅拷贝后的新对象将不会复制这些嵌套对象，而是仅仅复制了它们的引用(即内存地址)。这意味着，新对象和原始对象会共享这些嵌套层的可变对象。因此，对通过浅拷贝得到的对象中的嵌套层进行修改，也会影响到原始对象中的相应部分。

## 深拷贝

在Python中，深拷贝是指创建一个新的对象，并且递归地复制原对象中包含的对象，直到所有的对象都被复制。这样，原对象和深拷贝后的对象在内存中是完全独立的，对深拷贝对象的修改不会影响到原对象。

# 可变类型 vs. 不可变类型

在Python中，数据类型根据其创建后能否被修改分为**可变类型（Mutable）**和**不可变类型（Immutable）**两类。

## 可变类型

对象创建后，其内容可以被修改（如增删改），且内存地址（ID）保持不变。

数据类型：

- 列表（`list`）
- 字典（`dict`）
- 集合（`set`）
- 字节数组（`bytearray`）

## 不可变类型

对象创建后，内容无法被修改。任何修改操作都会生成新对象，原对象保持不变。

数据类型：

- 数字类型：整数（`int`）、浮点数（`float`）、复数（`complex`）、布尔（`bool`）
- 字符串（`str`）
- 元组（`tuple`）
- 冻结集合（`frozenset`）
- 字节（`bytes`）

## 核心区别

- **可变类型**：支持原地修改，适用于需要频繁更新的场景（如动态数据存储）。
- **不可变类型**：安全性高，可哈希（可作为字典键或集合元素），适用于常量数据或多线程环境
