# 语法格式

Python 使用 `try-except` 语句来处理异常，语法格式如下：

```python
try:
     尝试执行的代码块
except 异常类型 as 变量:
    检测到异常要执行的代码块
else:
    没有检测到异常要执行的代码块
finally:
    无论是否检测到异常都会执行的代码块
```

示例：

```python
try:
    # 尝试将用户输入的字符串转换成整型数据
    num1 = int(input("请输入一个数字："))
except:
    print("转换失败")
```

# 异常触发原理

当Python解释器执行代码时，会维护一个调用栈。当异常发生时：

1. 解释器停止当前代码执行
2. 沿调用栈向上回溯寻找异常处理器（exception handler）
3. 若到达顶层仍未处理，触发默认处理器（打印Traceback）

```python
def func1():
    raise ValueError("Deep error")
 
def func2():
    func1()
 
def main():
    try:
        func2()
    except ValueError as e:
        print(f"Caught exception: {e}")
 
main()  # 输出：Caught exception: Deep erro
```

# 常见错误类型

- `TypeError`：类型错误 — 比如，把字符串和整数相加，类型不匹配
- `ValueError`：值错误 — 比如，`int('abc')` 把非数字字符串转换为整数，值不合法
- `NameError`：名字错误 — 使用了未定义的变量
- `IndexError`：索引错误 — 列表或字符串索引超出范围
- `KeyError`：键错误 — 字典中访问不存在的键
- `FileNotFoundError`：文件未找到错误 — 尝试打开不存在的文件
- `IOError`：输入输出错误 — 文件读写过程中发生错误
- `ZeroDivisionError`：除零错误 — 除数为零
- `AttributeError`：属性错误 — 访问对象不存在的属性或方法

# 捕获异常描述信息

除此之外，我们还可以声明捕获 Exception ，它是一个万能异常类，能够捕获程序中发生的任意非语法错误类型的异常，并允许我们获取异常的具体描述信息。

```python
try:
    print(name)
except (
    Exception
) as e:  # 捕获程序中可能发生的任意非语法错误类型，并将捕获到的异常对象（包含异常描述信息）保存到变量 e 中
    print(f"异常具体描述信息：{e}")  # 获取异常的具体描述信息，以便进行进一步的处理

"""
异常具体描述信息：name 'name' is not defined
"""
```

# `else` & `finally`

## `else`

当 try 中的代码块成功执行且没有引发任何异常时，你希望执行一些额外的代码，此时可以放入 else 语句块中 

示例：

```python
try:
    # 尝试将用户输入的字符串转换为整型数据
    num = int(input("请输入一个数："))
except Exception as e:
    print(e)
else:
    print("转换成功")
    print(num)
```

## `finally`

`finally` 语句用于定义无论是否发生异常都要执行的代码块。它通常与 `try` 和 `except` 语句配合使用，以确保即使在遇到错误时也能执行某些操作，例如关闭文件、释放系统资源等。

示例：

 ```python
 print("打开文件")
 try:
     print("文件操作")
 except Exception as e:
     print(e)
 else:
     print("操作成功")
 finally:
     print("关闭文件")
 ```

> **注意**：`try-finally` 结构可以单独使用，也可以与 `extept` 和 `else` 结合使用，以提供灵活的异常处理机制。

 ```python
 try:
     print("文件操作")
 finally:
     print("关闭文件")
 ```

# 抛出自定义异常

在 Python中，如果你想要在自己的程序中主动抛出一个异常，可以使用 `raise` 语句来实现。`raise` 语句允许你指定要抛出的异常类型和可选的异常信息。

步骤如下：

1. 创建一个异常对象

   ```python
   异常类型(异常具体描述信息)
   ```

2. 使用 `raise` 语句抛出异常对象

Python 标准库中并未涵盖所有可能的异常类型，特别是那些与特定应用逻辑紧密相关的异常，例如 “密码不足8位数” 或 ”用户名不能为空等。针对这类情况，我们可以自定义异常，以便更精确地描述和处理程序中的错误情况。这样的自定义异常能够增强代码的可读性和可维护性。

需求： 用户输入密码，如果密码长度不足8位数，则系统主动抛出异常。

```python
# 定义函数
def input_pwd():
    # 提示用户输入密码
    pwd = input("请输入密码：")
    # 判断密码长度是否不足8位
    if len(pwd) < 8:
        # 抛出自定义异常
        raise Exception("密码长度不足 8 位数")
    print("密码输入成功")
    return pwd

# 为了防止程序因异常而终止，对异常进行捕获处理
try:
    # 调用函数并处处返回值
    print(input_pwd())
except Exception as e:
    # 输出异常具体信息
    print(e)

print("...")
```
