# 多任务

## 含义

现实生活中有很多场景中的事情是同时进行的，比如开车的时候手和脚共同来驾驶汽车，再比如唱歌跳舞也是同时进行的，试想，如果把唱歌和跳舞这两件事情分开依次完成的话，估计就没有那么好的效果了（想一下场景：先唱歌，然后在跳舞）。

> **思考**：利用现学知识能够让两个函数或者方法同时执行吗?

示例：

```python
# 导入 time 模块
import time


# 定义函数
def sing():
    print("我在唱歌")
    time.sleep(2)  # 暂停2秒
    print("唱完了")


def dance():
    print("我在跳舞")
    time.sleep(2)  # 暂停2秒
    print("跳完了")


sing()
dance()

"""
我在唱歌
唱完了
我在跳舞
跳完了
"""
```

由此可以发现，利用现学知识不能让两个函数同时执行，因为之前所写的程序都是单任务的，也就是说一个函数或者方法执行完成，另外一个函数或者方法才能执行，要想实现同时执行的功能就需要使用多任务。

**多任务就是指在同一时间段内执行多个任务。它的最大好处是充分利用CPU资源，提高程序的执行效率。**

## 实现方式

 在 Python 中，实现多任务处理主要有三种方式：

1. 线程
2. 进程
3. 协程

**进程 vs. 线程**

- 进程：打开一个程序至少就会有一个进程，是操作系统进行资源分配的基本单位
- 线程：线程是CPU调度的基本单位，每个进程至少都有一个线程，这个线程通常就是我们所说的**主线程**

一个进程默认有一个线程，进程里面可以创建多个线程，线程是依附在进程里面的，没有进程就没有线程。

# 多线程

## 含义

允许程序在同一时间内运行多个线程，是Python中实现多任务的一种方式。

注意：需导入线程模块

```python
# 导入多线程模块
import threading
```

## 线程类 Thread 常见参数

1. `target`：指定子线程要执行的任务，即要运行的函数。
2. `args`：以元组的形式给执行任务传参
3. `kwargs`:：以字典的形式给执行任务传参

## 多线程完成多任务

代码示例：

```python
import threading


# 任务 1：唱歌
def sing():
    print("我在唱歌")
    time.sleep(2)  # 暂停2秒
    print("唱完了")


# 任务2：跳舞
def dance():
    print("我在跳舞")
    time.sleep(2)  # 暂停2秒
    print("跳完了")


# 主程序入口
if __name__ == "__main__":
    # 创建线程对象(注意：子线程个数没有限制，而且子线程之间相互独立，互不影响)
    t1 = threading.Thread(target=sing)  # 唱歌
    t2 = threading.Thread(target=dance)  # 跳舞

    print(t1)
    print(t2)
    # 启动线程
    t1.start()
    t2.start()

    # 等待线程结束
    t1.join()
    t2.join()

    print("所有任务完成")
```

## 线程执行带有参数的任务

前面我们使用线程执行的任务是没有参数的，假如我们使用线程执行的任务带有参数，如何给函数传参呢?

`Thread` 类执行任务并给任务传参数有两种方式：

- `args`：表示以元组的方式给执行任务传参，必须要和参数顺序保持一致。
- `kwargs`：表示以字典的方式给执行任务传参，字典中的 `key` 必须要和参数名保持一致。

示例：

```python
# 导入 time 模块
import time

# 导入多线程模块
import threading


# 任务 1：唱歌
def sing(name):
    print(f"{name}在唱歌")
    time.sleep(2)  # 暂停2秒
    print("唱完了")


# 任务2：跳舞
def dance(name):
    print(f"{name}在跳舞")
    time.sleep(2)  # 暂停2秒
    print("跳完了")


# 主程序入口
if __name__ == "__main__":
    # 创建子线程

    t1 = threading.Thread(target=sing, args=("张三",))  # 以元祖形式为任务传参
    t2 = threading.Thread(target=dance, kwargs={"name": "李四"})  # 以字典形式为任务传参

    # 启动线程
    t1.start()
    t2.start()
```

## 多线程的注意点

### 线程之间执行是无序的

线程之间执行是无序的，它是由CPU调度决定的，CPU调度哪个线程，哪个线程就先执行，没有调度的线程不能执。

代码示例：

```python
# 导入 time 模块
import time

# 导入多线程模块
import threading


# 任务
def task():
    time.sleep(1)
    print(f"当前线程是：{threading.current_thread().name}")  # 显示当前线程名称


# 主程序入口
if __name__ == "__main__":
    for i in range(5):
        # 创建子线程
        t = threading.Thread(target=task, name=f"Thread-{i}")
        # 启动线程
        t.start()

"""
当前线程是：Thread-0
当前线程是：Thread-3
当前线程是：Thread-4
当前线程是：Thread-1
当前线程是：Thread-2
"""
```

### 线程之间共享资源（全局变量）

代码示例：

```python
# 导入 time 模块
import time

# 从线程模块中导入 Thread 类
from threading import Thread


# 全局变量
li = []


# 写入数据
def write_data():
    for i in range(5):
        li.append(i)
        time.sleep(1)  # 暂停 1 秒
    print(f"写入的数据是：{li}")


# 读取数据
def read_data():
    print(f"读取的数据是：{li}")


# 主程序入口
if __name__ == "__main__":
    t1 = Thread(target=write_data)  # 创建线程 t1，目标函数为 write_data
    t2 = Thread(target=read_data)   # 创建线程 t2，目标函数为 read_data

    t1.start()  # 启动线程 t1
    t1.join()   # 阻塞主线程：主线程等待写入线程执行完成以后代码再继续往下执行

    t2.start()  # 启动线程 t2
    t2.join()   # 阻塞主线程：主线程等待读取线程执行完成以后代码再继续往下执行

    print("Ended...")

```

### 资源竞争

当多个线程试图同时访问或修改同一资源时，就会产生竞争现象，这类似于多个男生同时追求同一个女生，存在激烈的竞争关系。

代码示例：

```python
# 导入 time 模块
import time

# 从线程模块中导入 Thread 类
from threading import Thread


# 全局变量
a = 0


def add1():
    for i in range(1000000):
        global a
        a += 1
    print(f"第一次: {a}")


def add2():
    for i in range(1000000):
        global a
        a += 1
    print(f"第二次: {a}")


# add1()
# add2()

"""
第一次: 1000000
第二次: 2000000
"""


if __name__ == "__main__":
    # 创建子线程
    t1 = Thread(target=add1)
    t2 = Thread(target=add2)

    # 启动线程
    t1.start()
    t2.start()

'''
第一次: 1929322
第二次: 2000000
'''
```

# 线程同步

## 含义

前面所比喻的资源竞争，就像多个男生同时追求同一个女生，这样的竞争导致结果难以预测。为了确保每个追求者都有公平的机会与女生相处，我们需要引入一种协调机制，这就是线程同步。

以两个线程为例：线程A负责写入数据，而线程B负责读取线程A所写入的数据。在这种情境下，为了确保数据的准确性和一致性，必须确保线程A完成写入操作后，线程B才能进行读取。这种确保线程A和线程B按序、协调执行的关系，就是我们所说的线程同步。通过线程同步，我们能够避免数据竞争和不一致的问题，从而确保程序的稳定性和可靠性。

## 方式

1. 线程等待 `join`
2. 互斥锁

示例：

```python
# 导入 time 模块
import time

# 从线程模块中导入 Thread 类
from threading import Thread


# 全局变量
a = 0


def add1():
    for i in range(1000000):
        global a
        a += 1
    print(f"第一次: {a}")


def add2():
    for i in range(1000000):
        global a
        a += 1
    print(f"第二次: {a}")


# add1()
# add2()

"""
第一次: 1000000
第二次: 2000000
"""


if __name__ == "__main__":
    # 创建子线程
    t1 = Thread(target=add1)
    t2 = Thread(target=add2)

    # 启动线程 t1
    t1.start()

    # 线程等待 join
    t1.join()

    # 启动线程 t2
    t2.start()
```

# 互斥锁

## 含义

对共享数据进行锁定是一种确保多线程环境下数据访问正确性的有效手段。这种锁定机制能够防止多个线程在同一时刻对共享数据进行操作，从而避免数据错误或不一致的问题。它确保了同一时间仅有一个线程能够操作共享数据，类似于将一个小姑娘置于一间上锁的房间内，男生们需依次进入，前一位男生离开并解锁后，下一位男生才能进入。这里的锁即为互斥锁。

值得注意的是，互斥锁并非静态分配，而是由多个线程动态竞争获取。成功获取锁的线程将先行执行相关操作，而未获取锁的线程则需耐心等待。直到持有锁的线程释放锁后，其他等待的线程才有机会再次竞争这把锁，进而执行自己的操作。这种机制确保了多线程环境下的数据访问既安全又有序。

## 方法

在 Python 的 `threading` 模块中，定义了 `Lock` 类。通过实例化这个类，我们可以获得一个互斥锁对象，这个对象提供了两个关键的方法：`acquire()` 和 `release()`。

- `acquire` 方法用于获取锁。如果锁当前被其他线程持有，则调用此方法的线程将被阻塞，直到锁被释放为止。在某些情况下，你也可以为 `acquire` 方法设置一个超时时间，以避免无限期地等待锁。
- `release` 方法用于释放锁。持有锁的线程调用此方法后，其他被阻塞、正在等待此锁的线程将有机会获取锁。

示例：

```python
# 导入 time 模块
import time

# 从线程模块中导入 Thread 类
from threading import Thread, Lock


# 全局变量
a = 0

# 创建全局互斥锁
lock = Lock()


def add1():
    # 上锁
    lock.acquire()
    for i in range(1000000):
        global a
        a += 1
    print(f"第一次: {a}")
    # 解锁
    lock.release()


def add2():
    # 上锁
    lock.acquire()
    for i in range(1000000):
        global a
        a += 1
    print(f"第二次: {a}")
    # 解锁
    lock.release()


if __name__ == "__main__":
    # 创建子线程
    t1 = Thread(target=add1)
    t2 = Thread(target=add2)

    # 启动线程
    t1.start()
    t2.start()

"""
第一次: 1000000
第二次: 2000000
"""
```

注意：如果互斥锁使用不当，特别是在多线程共享多个资源的情况下，就可能导致死锁问题。

- **死锁**：是指两个或多个线程相互等待对方释放锁，从而导致这些线程都无法继续执行的情况。这种场景类似于两个人在争吵后都坚持等待对方先道歉，结果导致双方都无法前进。在编程中，死锁会导致应用程序陷入停滞状态，无法继续处理其他任务。

因此，在使用互斥锁时，必须在适当的位置释放锁。一旦线程完成了对共享资源的访问，就应该立即释放锁，以便其他线程能够获取锁并访问该资源。

总结：在任何情况下，上锁操作后面都必须有对应的释放锁操作。这是确保锁能够被正确释放的关键。如果上锁后没有相应的释放锁操作，那么其他等待锁的线程将会一直等待下去，最终导致死锁。
