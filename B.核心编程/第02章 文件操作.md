# 文件

## 含义

存储在某种长期储存设备(如硬盘、U盘、移动硬盘等)上的一段数据，用于持久化保存信息。

## 作用

将内容（数据）进行存储和存放，以便程序在需要时能够直接访问和使用这些数据，而不需要重新生成或制作一份新的数据。这种做法能够节省时间和资源，提高程序的效率和便捷性。

## 存储方式

计算机内部所有的数据都是以二进制形式存储和处理的，因此磁盘上的每一个文件本质上都是二进制数据。但是在实 际应用中，根据文件的内容和功能，我们可以将其分为文本文件和二进制文件两大类。

1. **文本文件**
   - **内容**：主要存储人类可读的字符串信息，如源代码、文档等。这些字符信息按照特定的字符编码（如 `ASCII`、`UTF-8` 等）被转换为二进制数据存储在文件中。
   - **查看方式**：由于文本文件的内容是人类可读的，因此可以使用文本编辑软件（如 Windows 的记事本）进行查看和编辑。
   - **文件扩展名**：常见的文本文件扩展名包括 `.txt`（纯文本）、`.py`（Python 源代码文件）等。
   - **注意**：尽管文本文件的内容对人类来说是可读的，但它们在计算机内部仍然是以二进制形式存储的。
2. **二进制文件**
   - **内容**：直接以二进制形式存储，包含的数据类型更加广泛，如图片、视频、音频、可执行程序等。这些数据通常不是为人类直接阅读而设计的，而是为了计算机程序或特定软件能够高效处理和利用。
   - **查看方式**：由于二进制文件的内容不是以人类可读的字符形式组织的，因此通常无法使用文本编辑软件直接查看其内容。需要使用专门的软件或者工具（如图片查看器、视频播放器、音频播放器）来解读和处理这些文件。
   - **文件扩展名**：二进制文件的扩展名多种多样，取决于文件的具体类型和用途，如：`.jpg`（图片文件）、`.mp3`（音频文件）、`.exe`（Windows 可执行程序文件） 等。

> 总结：无论是文本文件还是二进制文件，在计算机中都是以二进制形式存储的。它们的主要区别在于内容、用途以及人类如何查看和处理它们。

## 绝对路径 vs. 相对路径

相对路径和绝对路径是计算机中用于定位文件或目录的两种不同方式

1. **绝对路径（Absolute Path）**
   - **定义**：从根目录（如 `C:\` 或 `/`）开始的完整路径，直接指向目标文件或目录的位置
   - **特点**：
     - 以盘符（Windows）或 `/`（Linux/Unix）开头。
     - 唯一且明确，与当前目录无关。
     - 示例：
       - Windows：`C:\Users\Document\file.txt`
       - Linux：`/home/user/Documents/file.txt`
       - Web：`https://example.com/images/logo.png`
2. **相对路径（Relative Path）**
   - **定义**：以当前工作目录为基准的路径，通过相对位置指向目标文件或目录
   - **特点**：
     - 不以根目录开头，依赖当前目录。
     - 使用特殊符号简化路径：
       - `.` 表示当前目录（如 `./file.txt`）
       - `..` 表示上一级目录（如 `../parent/file.txt`）
     - 示例：
       - 当前目录为 `/home/user`，目标文件为 `/home/user/Documents/file.txt`，则相对路径为 `Documents/file.txt`

实际应用建议：

- **绝对路径**：适合需要明确路径的场景（如脚本、外部资源引用）
- **相对路径**：适合项目内部文件引用，便于迁移和协作（如 `css/style.css`）。

# 基础操作

在计算机中，对文件的操作通常遵循一套固定的步骤，主要包括以下三个基本操作:

1. 打开文件：使用 `open` 函数，返回一个文件对象，默认以只读模式打开。
2. 读/写文件
   - **读**：将文件内容从磁盘加载到内存中，以便程序可以处理这些数据(因为内存的读写速度远快于磁盘)
   - **写**：将内存中的数据写入到磁盘文件中，实现数据的持久化存储。
3. 关闭文件：使用 `close` 方法关闭文件，释放系统资源。

> 注意：可以只打开和关闭文件，而不进行任何读写操作，但这通常不是文件的典型使用场景。

> 📖 **文件对象的常见方法**

1. `open()`：创建一个File（文件）对象，默认以只读模式打开。
2. `read(n)`：从文件中读取 `n` 个字符（在文本模式下）或 `n` 个字节（在二进制模式下）。如果未指定 `n`，则读取整个文件内容。
3. `write(s)`：将置顶内容 `s` 写入文件，在写入之前，需要将数据转换为字符串（在文本模式下）或字节串（在二进制模式下）。
4. `close()`：关闭文件。这是一个非常重要的方法，用于在完成文件操作后释放文件资源。

在当前项目下新建一个 `test.txt` 文件，内容个如下：

```txt
四川·成都
张三
30
```

需求：编写一个 Python 程序，打开并随后关闭 `test.txt` 文件。

代码实现：

```python
# 打开文件
f = open("test.txt")

print(f)  # <_io.TextIOWrapper name='test.txt' mode='r' encoding='UTF-8'>

# 关闭文件
f.close()
```

> **注意**：为了确保系统资源的有效利用和避免潜在的访问冲突，打开文件后务必及时关闭。文件若未关闭，将持续占用系统资源，而操作系统对同时打开的文件数量有限制，这不仅可能耗尽资源，还可能阻碍或干扰后续对文件的正常访问和操作。

> 📖 **文件对象的常见属性**

1. `file.name`：返回文件对象的名称（相对路径/绝对路径）
2. `file.mode`：返回文件的访问模式
3. `file.closed`：返回一个布尔值，指示文件是否已关闭

代码示例：

```python
# 打开文件
f = open("test.txt")

# 文件属性
print(f)  # <_io.TextIOWrapper name='test.txt' mode='r' encoding='UTF-8'>

print(f.name)  # test.txt
print(f.mode)  # r
print(f.encoding)  # UTF-8
print(f.closed)  # False

# 关闭文件
f.close()
```

## 打开文件

语法形式：

```python
f = open(workfile, mode, encoding="utf-8")
```

参数解读：

- `workfile`：文件名

- `mode`，模式：
  - 基础模式
    - `'r'`：**只读**（默认）。文件必须存在，否则报错 `FileNotFoundError`。等同于 `'rt'`（文本模式读取） ✔️
    - `'w'`：**只写**。覆盖同名文件内容；文件不存在则创建。等同于 `'wt'`（文本模式写入） ✔️
    - `'a'`：**追加**。写入内容自动加至文件末尾；文件不存在则创建，不允许读取。✔️
    - `'x'`：**独占创建**。文件存在则报错 `FileExistsError`；不存在则创建。等同于 `'xt'`
    - `'b'`：**二进制模式**。需与其他模式组合（如 `'rb'`、`'wb'`），直接操作字节数据
    - `'t'`：**文本模式**（默认）。与 `'r'`/`'w'`/`'a'`/`'x'` 组合时通常省略（如 `'rt'` ≡ `'r'`）
  - 组合模式
    - `'+'`：**读写模式**。需与其他模式组合：
      - `'r+'`：读写（文件必须存在，指针在开头）✔️
      - `'w+'`：读写（覆盖同名文件；不存在则创建）✔️
      - `'a+'`：读写（追加写入，指针默认在末尾）✔️
      - `'x+'`：独占创建读写（文件存在则失败）
    
    > **注意**：使用 `+` 会影响文件的读写效率，开发中更多时候会以只读、只写的方式来操作文件。
  - 二进制扩展
    - `'rb'`：二进制只读
    - `'wb'`：二进制只写（覆盖）
    - `'ab'`：二进制追加
    - `'r+b'`/`'rb+'`：二进制读写
    - `'w+b'`/`'wb+'`：二进制读写（覆盖）
    - `'a+b'`/`'ab+'`：二进制读写（追加）
  
- `encoding`：编码格式，如果省略，则使用平台默认编码，如 Windows 系统上默认编码为 `GBK`，Linux 系统默认编码为 `utf-8`。建议明确指定编码格式为 `utf-8`，避免出现乱码。

代码示例：

```python
# 打开文件
f = open("test.txt")

# 文件属性
print(f)  # <_io.TextIOWrapper name='test.txt' mode='r' encoding='UTF-8'>

print(f.name)  # test.txt
print(f.mode)  # r
print(f.encoding)  # UTF-8
print(f.closed)  # False

# 关闭文件
f.close()
```

## 关闭文件

在处理文件对象时，推荐使用 [`with`](https://docs.python.org/zh-cn/3/reference/compound_stmts.html#with) 关键字自动管理文件关闭。优点是，子句体结束后，文件会正确关闭，即便触发异常也可以。而且，使用 `with` 相比等效的 [`try`](https://docs.python.org/zh-cn/3/reference/compound_stmts.html#try)-[`finally`](https://docs.python.org/zh-cn/3/reference/compound_stmts.html#finally) 代码块要简短得多：

```python
with open("example.json", "r", encoding="utf-8") as file:
    data = file.read()
    print(data)
    
print(file.closed)  # True
```

如果没有使用 [`with`](https://docs.python.org/zh-cn/3/reference/compound_stmts.html#with) 关键字，则应调用 `f.close()` 关闭文件，即可释放文件占用的系统资源。

```python
# 打开文件
with open("example.json", "r", encoding="utf-8") as file:
    # 读取文件
    data = file.read()
    print(data)

# 检测文件关闭状态
print(file.closed)  # True
```

> 警告：调用 `f.write()` 时，未使用 `with` 关键字，或未调用 `f.close()`，即使程序正常退出，也**可能** 导致 `f.write()` 的参数没有完全写入磁盘。

通过 [`with`](https://docs.python.org/zh-cn/3/reference/compound_stmts.html#with) 语句，或调用 `f.close()` 关闭文件对象后，再次使用该文件对象将会失败。

```python
file.close()
file.read()  # ValueError: I/O operation on closed file.
```

# 读取文件

## `read(n)`

从文件中读取 `n` 个字符（在文本模式下）或 `n` 个字节（在二进制模式下）。如果未指定 `n`，则读取整个文件内容。

> `test.txt`

```tex
PythonStudy
```

代码示例：

```python
with open("test.txt") as f:
    print(f.read())   # 默认读取全部内容 输出：··
    print(f.read(6))  # 读取6个字符 输出：Python
```

使用 `read` 方法读取文件时，会将文件的所有内容一次性加载到内存中，这对于非常大的文件来说，可能会给系统内存带来沉重的压力。为了更有效地管理内存资源，推荐在处理大文件时采用 `readline` 方法，以便逐行读取文件内容，减少内存占用。

## `readline()`

从文件中读取一行内容。读取完一行后，文件指针会自动移动到下一行的开头，准备下一次读取。

`test.txt`

```tex
张三
30
四川省成都市
```

代码示例：

```python
with open("test.txt") as f:
    print(f.readline())  # 读取第一行
    print(f.readline())  # 读取第二行
    print(f.readline())  # 读取第三行
    print(f.readline())  # 读取第四行，单文件只有三行，所以返回空字符

"""
张三
30
四川省成都市

"""
```

优化版本：

```python
# 循环次数未知，结束条件已知 ==> 死循环 ==> while True
with open("test.txt") as f:
    while True:
        text = f.readline()
        # 判断读取内容是否为空（字符）
        if text == "":
            break
        # 输出读取内容
        print(text)
```

## `readlines()`

从文件中读取所有行，并将这些行作为一个列表返回，列表中的每个元素都是文件中的一行。

代码示例：

```python
with open("test.txt") as f:
    text = f.readlines()
    print(text, type(text))  # ['张三\n', '30\n', '四川省成都市'] <class 'list'>

    # 遍历列表
    for row in text:
        print(row.strip())

"""
张三
30
四川省成都市
"""
```

> **注意**：同样地，如果文件非常大，使用 `readlines` 方法可能会消耗大量的内存，因为它需要将整个文件内容都加载到内存中。

## 分块读取

处理大文件时避免内存溢出

```python
with open('large_file.log', 'r') as f:  # ① 打开文件
    while chunk := f.read(1024):        # ② 分块读取
        process(chunk)                  # ③ 处理数据块
```

# 写入文件

在 Python 中，向文件写入内容时，需要注意文件的访问模式，也就是调用 `open` 时设置的 `mode` 参数。

```python
with open("test.txt") as f:
    # 读取操作
    print(f.read())
    # 写入操作
    f.write("yyds")  # io.UnsupportedOperation: not writable
```

在这个例子中，默认以只读模式打开 `test.txt` 文件，在只读模式下，文件不能被写入。因此尝试向该文件写入内会导致 `io.UnsupportedOperation` 异常。为了解决这个问题，需要更改访问模式来打开文件以进行写入。

写入文件的常用方法：

- `write()`：写入字符串。
- `writelines()`：写入字符串列表
- 追加模式：`'a'` 模式保留原内容

```python
with open('output.txt', 'w') as file:
    file.writelines(["Line1\n", "Line2\n"])
```

# 文件定位操作

1. `tell()`：返回文件指针的当前位置
2. `seek(offset, whence=0)`：移动文件指针到指定位置
   - `offset`：偏移量，表示要移动的字节数
   - `whence`：用于指定 `offset` 的起始位置(参考点)，默认值是 `0`。
     - `0` 代表将文件开头作为参考位置
     - `1` 代表将当前位置作为参考位置
     - `2` 代表将文件结尾作为参考位置。

**注意**：通常使用 `seek(0, 0)` 以将文件指针移动到文件开头。

代码示例：

```python
with open("test.txt", "a+") as f:
    # 获取移动前文件指针当前的位置
    print(f"移动前：{f.tell()}")

    # 移动文件指针到文件开头
    f.seek(0, 0)

    print(f"移动后：{f.tell()}")

    print(f"读取内容：{f.read()}")
```

# 目录操作

## `os` 模块

### 基础操作

- 文件或目录重命名：`os.rename(src, dst)`
- 删除文件：`os.remove(path)`
- 创建单级目录：`os.mkdir(path, mode=0o777)`
- 创建多级目录：`os.makedirs(path, mode=0o777, exist_ok=False)`
- 删除空目录：`os.rmdir(path)`
- 递归删除目录树：`shutil.rmtree(path)`
- 获取当前工作目录：`os.getcwd()`
- 更改当前工作目录：`os.chdir(path)`

### 目录遍历与查询

- 获取目录列表：`os.listdir(path)`
- 递归遍历目录树：`os.walk(top, topdown=True, onerror=None)`
- 高效目录遍历：`os.scandir(path)`
- 检查路径是否存在：`os.path.exists(path)`
- 检查是否为文件：`os.path.isfile(path)`
- 检查是否为目录：`os.path.isdir(path)`

### 路径处理

- 路径拼接：`os.path.join(path1, path2, ...)`
- 获取绝对路径：`os.path.abspath(path)`
- 获取文件名：`os.path.basename(path)`
- 获取目录名：`os.path.dirname(path)`
- 分割扩展名：`os.path.splitext(path)`
- 获取文件大小：`os.path.getsize(path)`

### 高级操作

- 递归重命名：`os.renames(old, new)`
- 递归删除空目录：`os.removedirs(path)`
- 复制目录树：`shutil.copytree(src, dst)`
- 移动目录/文件：`shutil.move(src, dst)`

## `pathlib` 模块

- 创建Path对象：`Path(path)`
- 创建目录：`Path.mkdir(mode=0o777, parents=False, exist_ok=False)`
- 删除目录：`Path.rmdir()`
- 遍历目录：`Path.iterdir()`
- 通配符搜索：`Path.glob(pattern)`
- 递归搜索：`Path.rglob(pattern)`

# 案例：图片备份（复制）

之前讲解侧重于文本文件的操作，然而，在处理诸如图片、视频等二进制文件时，我们必须在访问模式字符串中明确加入 `"b"` 标志(如使用 `"rb"` 进行读取、`"wb"` 进行写入等)，以此指示 Python 以二进制模式执行操作。这样做确保了Python 能够直接按照文件的原始字节流进行处理，避免了不必要的字符编码转换，从而实现了对二进制数据的正确且高效管理。

示例：

```python
# 打开源文件
with open("logo.jpg", "rb") as f:
    # 读取文件内容
    content = f.read()
    print(content, type(content))  # ... <class 'bytes'>

# 将读取到的内容感谢如到新文件中
with open("image[备份].jpg", "wb") as f:  # 打开不存在的文件 ==> 创建新文件
    # 写入文件内容
    f.write(content)
    print("写入成功")
```
