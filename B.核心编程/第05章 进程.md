# 进程介绍

## 含义

一个正在运行的程序或者软件就是一个进程，它 **是操作系统进行资源分配和调度的基本单位**。也就是说每启动一个进程，操作系统都会给其分配一定的运行资源（如内存、网络连接等）保证进程的运行。

注意：**一个程序运行后至少会生成一个进程，一个进程默认包含一个线程**，进程里面可以创建多个线程，**线程是依附在进程里面的，没有进程就没有线程**。

## 进程的状态

工作中，任务数往往大于CPU的核数，即一定有一些任务正在执行，而另外一些任务在等待CPU进行执行，因此进程会经历不同的状态。这些状态包括：

- 就绪态：已经具备执行条件，正在等待CPU的调度以开始执行。例如，一个已经加载到内存中并准备好执行的计算器程序，它正在等待CPU分配空闲时间片来启动计算任务。
- 执行态：CPU正在执行其功能。例如，一个正在播放视频的进程，此时CPU正在处理视频帧的解码和渲染工作，以确保视频能够流畅播放。
- 等待态：正在等待某个事件或条件的发生。例如，一个网页浏览器进程，可能正在等待用户输入(如点击按钮)来继续执行后续操作。

除了上述三种基本状态外，进程还可能经历其他状态，如挂起态和终止态：

- 挂起态：被操作系统暂时停止执行，这通常是为了节省内存资源或等待外部事件。例如，当手机中的一个应用程序在后台运行时，它可能会被挂起，以释放CPU和内存资源给前台运行的应用程序。当用户再次切换到该应用程序时，它会被恢复执行。
- 终止态：已经执行完毕或被操作系统强制终止。例如，一个计算器程序进程在完成计算任务并输出结果后可能会进入终止态，或者，一个因为崩溃而被操作系统强制关闭的游戏进程也会进入终止态。此时，进程会释放其占用的所有资源，以供其他进程使用。

# 多进程

`multiprocessing`  模块是一个跨平台的多进程管理模块，它定义了 `Process` 类来创建并管理独立的进程对象，这些进程对象能够并行地执行不同的任务。

## 进程语法结构

- Process 常见参数
  - `target`：指定子进程要执行的任务，即要运行的函数。
  - `args`：以元组的形式给执行任务传参
  - `kwargs`：以字典的形式给执行任务传参
- Process 常见方法
  - `start()`：开启子进程。
  - `is_alive()`：检查子进程是否仍然在运行。如果子进程仍然在运行，则返回 `True`，如果子进程已经终止，则返回 `False`。
  - `join()`：阻塞主进程，即主进程等待子进程执行结束
- Process 常见属性
  - `name`：当前进程的别名。
  - `pid`：当前进程的进程编号。

代码示例：

```python
# 导入模块
from multiprocessing import Process
import os


# 唱歌任务
def sing(name):
    print(f"{name}在唱歌")
    # 访问子进程编号和父进程编号
    print(f"sing 子进程pid：{os.getpid()}，父进程的pid：{os.getppid()}")


# 跳舞任务
def dance(name):
    print(f"{name}在跳舞")
    print(f"dance 子进程pid：{os.getpid()}，父进程的pid：{os.getppid()}")


# 主程序入口
if __name__ == "__main__":
    # 创建子进程
    p1 = Process(target=sing, name="P1", args=("张三",))
    p2 = Process(target=dance, name="P2", kwargs={"name": "李四"})

    # 开启子进程
    p1.start()

    # 主进程等待子进程执行结束
    p1.join()

    p2.start()
    p2.join()

    # 修改子进程 name 属性
    p1.name = "Pro1"
    p2.name = "Pro2"

    # 访问 name 属性：子进程对象名.name
    print(f"子进程1的名字：{p1.name}")
    print(f"子进程2的名字：{p2.name}")

    # 访问子进程的 pid 属性
    print(f"子进程1的pid：{p1.pid}")
    print(f"子进程2的pid：{p2.pid}")

    # 查看主进程父进程编号
    print(f"主进程的pid：{os.getpid()}，父进程的pid：{os.getppid()}")

    # 检查子进程是否仍然在运行
    print(f"P1 的存活状态：{p1.is_alive()}")
    print(f"P2 的存活状态：{p2.is_alive()}")
```

# 进程间的通信

进程之间的资源不共享，如下所示：

```python
import time
from multiprocessing import Process

# 定义全局变量
li = []


# 定义任务：写入数据
def write_data():
    for i in range(5):
        li.append(i)
        time.sleep(1)
    print(f"写入的数据：{li}")


# 定义任务：读取数据
def read_data():
    print(f"读取的数据：{li}")


# 主程序入口
if __name__ == "__main__":
    # 创建子进程
    wp = Process(target=write_data)
    rp = Process(target=read_data)

    # 开启子进程
    wp.start()
    wp.join()  # 等数据写入完成之后再去读取

    rp.start()

"""
写入的数据：[0, 1, 2, 3, 4]
读取的数据：[]
"""
```

## 进程操作队列

由于进程之间默认不直接共享资源，当它们需要访问或操作相同的数据或资源时，就需要借助进程间通信机制来传递信息或数据，从而实现资源的共享与协调使用。

`multiprocessing` 模块中的 `Queue` 类是一个功能强大且易于使用的工具，它专为多进程环境而设计，能够实现进程间数据的安全传递。其常见方法有:

1. `put()`：放入数据(入队)。
2. `get()`：取出数据(出队)。
3. `empty()`：判断队列是否为空。如果为空，返回 `True` ，否则返回 `False`。
4. `qsize()`：返回当前队列包含的消息数量。

5. `full()`：判断队列是否已满。如果已满，返回 `True`，否则返回 `False`。 

代码示例1：

```python
import time
from multiprocessing import Process, Queue  # 进程模块里面自带 Queue 队列


# 实例化队列对象
q = Queue(3)  # 最多可接受 3 条消息，如果没写，或是负值，则表示没有上限

# 放入数据
q.put("今天平安夜")
q.put("要回家吃苹果了")
q.put("好开心哦")

# 判断消息是否已满
print(f"消息是否已满：{q.full()}")  # True

# 取出数据
print(q.get())  # 今天平安夜
print(q.get())  # 要回家吃苹果了


# 判断消息是否已满
print(f"消息是否已满：{q.full()}")  # False


# 判断队列是否为空
print(f"队列是否为空{q.empty()}")  # False


# 取出数据
print(q.get())  # 好开心哦

# 判断队列是否为空
print(f"队列是否为空{q.empty()}")  # True


"""
消息是否已满：True
今天平安夜
要回家吃苹果了
消息是否已满：False
队列是否为空False
好开心哦
队列是否为空True
"""

```

代码示例 2：

```python
import time
from multiprocessing import Process, Queue  # 进程模块里面自带 Queue 队列

# 定义全局变量
li = ["张三", "李四", "赵二", "王五"]


# 写入数据
def write_data(q):
    # 将列表中的数据依次放入队列当中
    for item in li:
        print(f"{item} 已经被放入")
        q.put(item)
        time.sleep(0.2)


# 读取数据
def read_data(q):
    # 只要队列中有消息就取出来
    while True:
        # 判断队列是否为空，如果为空就退出循环
        if q.empty():
            break
        # 不为空就取出消息
        else:
            print(f"从队列中取出数据：{q.get()}")


if __name__ == "__main__":
    # 避免 macOS 下 multiprocessing 重复导入导致的问题
    print("主程序开始")

    # 创建队列
    q = Queue()

    # 创建子进程
    wp = Process(target=write_data, args=(q,))
    rp = Process(target=read_data, args=(q,))

    wp.start()
    wp.join()  # 等写完再读

    rp.start()
    rp.join()

    print("主程序结束")

```

# 进程池

## 含义

前面我们仅仅只是创建了两个进程，但当需要创建几十个甚至上百个进程时，逐个实例化进程对象不仅操作繁琐，而且效率低下。为更有效地解决这一问题，我们可以采用进程池。

进程池可以被形象地比作一个水池，其中的“水盆”代表进程。在这个水池中，我们可以设定”水盆”的数量，即限制同时运行的进程数，从而确定能并行处理的最大任务数量。

具体而言，进程池会预先创建并维护一定数量的进程。当有任务需求时，进程池会分配一个空闲的进程来处理该任务。任务完成后，该进程不会立即关闭，而是回到进程池中继续等待新的任务分配。这种方式显著提高了资源利用率和任务处理效率。

## 常用方法

1. `apply_async()`：异步非阻塞地执行任务，这意味着主进程不会等待当前任务执行完成，而是根据系统调度来进行进程切换，从而可以处理其他任务或继续执行主进程的其他代码。

   > **注意**：如果使用异步提交任务，并且需要收集结果，应使用 `get`方法来获取，`get` 方法会阻塞，直到结果可用。

2. `c1ose()`：关闭进程池，使其不再接受新的任务。已经提交的任务会继续执行，但新的任务将无法提交到进程池中。

3. `join()`：阻塞主进程，直到进程池中的所有工作进程都退出，这通常用于等待所有任务都完成后再继续执行主进程的后续代码。

   > **注意**：`join` 方法必须在 `close` 方法之后调用，否则可能会引发异常或导致未定义行为。

代码示例：

```python
import time
from multiprocessing import Process, Queue, Pool


# 任务
def task(num):
    print("五一快到了，还在执行任务～")
    time.sleep(2)
    return num * 2


# 主程序入口
if __name__ == "__main__":
    # 实例化进程池，最大进程数是 3
    p = Pool(3)
    # 定义一个空列表
    li = []

    # 循环
    for i in range(7):
        # 执行：apply_async(调用的目标方法, 参数)
        # 异步：进程不需要一直等待下去，而是继续执行下面的操作，不管其他进程的状态
        res = p.apply_async(task, args=(i,))
        print(res)
        # 将返回的数值结果放进列表中
        li.append(res)

    # 关闭进程池
    p.close()
    # 等待进程池中所有子进程执行完毕，必须放在 close() 后面
    p.join()

    # 使用 get 方法来获取 apply_async() 的结果
    for i in li:
        print(i.get())
```
