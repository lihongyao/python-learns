# 协程介绍

## 含义

协程（Coroutine），又称微线程或纤程，是 Python 中一种轻量级的多任务实现方式。相较于线程，协程的执行单元更小、资源占用更低，因为它不需要像线程那样创建独立的内存空间和系统资源。协程自带CPU上下文切换的功能，允许开发者在适当的时候，灵活地在不同协程间进行切换。在切换过程中，通过保存和恢复CPU上下文，程序能够持续保持其运行状态，从而确保多任务处理的连续性和高效性。

简单来说，协程允许在单个线程内的函数执行过程中，随时保存当前函数的执行状态（包括临时变量等信息），并切换到另一个函数继续执行，这种切换并非通过函数调用实现，而是由开发者根据实际需求，自主决定切换的时机和次数。

**注意**：线程与进程的操作通常需要程序调用系统接口，并由操作系统负责最终的执行；而协程的调度与执行则完全由程序员掌控，无需依赖操作系统的介入。

## 简单实现协程

注意：生成器函数可以被用来简洁地实现协程的基本功能，即实现从一个函数到另一个函数的灵活切换。

```python
# 生成器函数：函数中有 yield 关键字
def task1():
    yield 123  # yield 作用：返回 123，并暂停函数，在此处挂起，使用 next() 再次调用生成器时，再从此处恢复执行
    yield "张三"


def task2():
    yield 456
    yield "李四"


# 主程序入口
if __name__ == "__main__":
    # 调用函数
    t1 = task1()  # 调用生成器函数，返回的是一个生成器对象
    t2 = task2()

    # 获取值
    # print(next(t1))  # 123
    # print(next(t2))  # 456

    # print(next(t1))  # 张三
    # print(next(t2))  # 李四
```

总结：通过简单的生成器函数，我们可以实现协程的基本功能，从而在处理多任务时，能够按照程序员的意愿灵活调整任务的执行顺序。在上面的示例中，通过控制 `next(t1)`和 `next(t2)` 的调用顺序，我们可以决定哪个任务的输出先被执行。

## 应用场景

1. 如果一个线程里面 I/O 操作（网络请求、文件操作）比较多，协程就比较适用。
2. 适合用于高并发处理。

# greenlet

`greenlet` 是一个第三方库，它提供了一种轻量级的协程实现。通过 switch 方法，它能够在不同的执行路径(或称为协程、greenlet)之间进行手动切换。然而，需要注意的是，由于这种切换是手动的，所以当某个协程遇到 I/O 操作时，如果没有额外的机制来处理，程序将会在该处阻塞，并不会自动切换到其他协程。

安装指令：

```shell
$ pip install greenlet
```

代码示例：

```python
# 导入模块
from greenlet import greenlet


# 唱歌任务
def sing():
    print("在唱歌")
    g2.switch()  # 唱完了没有执行，切换到 g2 中运行，需要在 g2 中再切换回来
    print("唱完了")


# 跳舞任务
def dance():
    print("在跳舞")
    # g1.switch()  # 在唱歌 - 在跳舞 - 唱完了
    print("跳完了")
    g1.switch()  # 在唱歌 - 在跳舞 - 跳完了 - 唱完了


# sing()
# dance()


# 主程序入口
if __name__ == "__main__":
    # 实例化一个协程对象：greenlet(任务名)
    g1 = greenlet(sing)
    g2 = greenlet(dance)

    g1.switch()  # 切换到 g1 中运行
    # g2.switch()  # dance() 就不会执行

```

`greenlet` 工作流程为：开发者需要手动通过 `switch` 方法在不同的执行路径(协程或 greenlet )之间进行切换。当某个 `greenlet` 遇到阻塞的 I/O 操作时，如果没有其它机制来主动处理这种阻塞，那么该 `greenlet` 的执行将暂停，导致程序在该处停滞。然而，`greenlet` 框架允许我们切换到另一个当前未被阻塞的 `greenlet` 继续执行，一旦原先阻塞的 `greenlet` 的阻塞状态被解除(例如网络请求完成)，开发者可以手动调用 `switch` 方法切换回原来的 `greenlet`，从阻塞点之后继续执行剩余的代码。

# gevent

## 含义

`greenlet` 虽然实现了协程的基本概念，但需要开发者手动通过 `switch` 方法在不同的执行路径之间进行切换，这在一定程度上增加了开发的复杂性和工作量。为了解决这个问题， `gevent` 这一功能更为强大且能够自动切换任务的第三方库应运而生。

在 `gevent` 中，当遇到 I/O 操作时，它会自动进行主动式任务切换，而 `greenlet` 则是其实现这一功能的主要协程模式。

在使用 `gevent` 之前，同样需要先通过 pip 命令来安装它，具体安装命令为： 

```shell
$ pip install gevent
```

## 常用方法

- `spawn()`：创建一个新的协程对象，并立即开始执行传入的函数。
- `sleep()`：执行一个耗时操作，即让当前协程暂停执行指定的秒数
- `join()`：阻塞当前协程，直到指定的协程对象执行完毕。
- `joinall()`：等待传入的协程对象列表中的所有协程都执行完毕。

示例1：

```python
import gevent


# 唱歌任务
def sing():
    print("在唱歌")
    gevent.sleep(1)  # 模拟的是 gevent 可识别的 I/O 阻塞
    print("唱完了")


# 跳舞任务
def dance():
    print("在跳舞")
    # gevent.sleep(1)
    gevent.sleep(2)  # 顺序一样，但时间差变了
    print("跳完了")


# 主程序入口
if __name__ == "__main__":
    # 创建协程对象
    g1 = gevent.spawn(sing)
    g2 = gevent.spawn(dance)

    # 阻塞，等待协程执行完毕
    g1.join()  # 等待 g1 对象执行结束
    g2.join()  # 等待 g2 对象执行结束

```

示例 2：

```python
import gevent
import time


# 唱歌任务
def sing(name):
    for i in range(1, 4):
        # gevent.sleep(1)
        time.sleep(1)
        print(f"{name}在唱歌，被送走的第{i}次")


# 主程序入口
if __name__ == "__main__":
    # 等待所有的协程都执行完毕
    gevent.joinall(
        [
            # 创建协程对象
            gevent.spawn(sing, "张三"),
            gevent.spawn(sing, "李四"),
        ]
    )


# 注意：如果没有遇到可识别的I/O操作，不会进行任务切换实现并发效果

```

## 给程序打补丁

在之前的 `gevent` 协程实现中，我们使用了 `gevent.sleep()` 来替代 `time.sleep()`以模拟等待，这是因为 `gevent` 需要通过其内部实现的模块来处理耗时操作，从而实现协程之间的自动切换。

然而，在实际开发中，代码往往更加复杂，直接修改代码并不总是可行的。为了解决这个问题，`gevent` 提供了一个强大的 `monkey` 补丁功能，该功能能够在程序运行时动态地替换模块中的函数或方法，从而将耗时代码自动转换为 `gevent` 的实现，而无需修改原始代码。

示例：

```python
import gevent
import time
from gevent import monkey


monkey.patch_all()  # 将用到的 time.sleep()，替换为 gevent 里面自己实现耗时操作的 gevent.sleep()
# 注意：monkey.patch_all() 必须放在被打补丁者的前面


# 唱歌任务
def sing(name):
    for i in range(1, 4):
        # gevent.sleep(1)
        time.sleep(1)
        print(f"{name}在唱歌，被送走的第{i}次")


# 主程序入口
if __name__ == "__main__":
    # 等待所有的协程都执行完毕
    gevent.joinall(
        [
            # 创建协程对象
            gevent.spawn(sing, "张三"),
            gevent.spawn(sing, "李四"),
        ]
    )


# 注意：如果没有遇到可识别的I/O操作，不会进行任务切换实现并发效果

```
