# 面相对象介绍

## 含义

是一种抽象化的编程思想，已成为当前软件开发领域的主流技术。

当前主流的编程思想中，以面向对象和面向过程两种最为显著。这两种编程思想各有其特点和适用场景 

## 面相过程 vs. 面相对象

### 面相过程

在学习面向对象编程之前，我们普遍采用面向过程的思维方式来解决问题。这一方式历史悠久，以其直观性和直接性，特别适合开发规模适中、结构清晰的中小型项目。在这一领域，C语言无疑是面向过程编程的杰出代表。

### 面相对象

接下来我们要学习的是面向对象编程，相较于面向过程而言出现较晚，特别适合构建复杂且规模庞大的项目。在这领域，典型代表有Python 和 Java 等。

### 区别

众所周知，洗衣服有两种方式：手洗与机洗。手洗过程繁琐，需经历找桶、装水、加洗衣粉、浸泡、搓洗、多次拧干与漂洗，直至最后晾晒，耗时费力。而机洗则截然不同，仅需简单几步 —— 开启洗衣机、放入衣物、添加洗涤剂、按下启动键，随后晾晒即可，操作极为便捷。这两种截然不同的洗衣方式，恰如编程中的面向过程与面向对象两种思维方式，各自以其独特的方式完成相同的任务。

**面向过程(手洗**)：在实现一个功能时，注重的是具体的执行步骤。将整个任务拆解成多个独立的步骤，并为每个步骤定义一个函数，然后通过依次调用这些函数来完成整个任务。这种方式下，每一步都需要我们亲自去定义和实现，如同手洗衣物，每一步都需要我们亲自动手操作。

**面向对象(机洗)**：在实现一个功能时，更侧重于"谁"来执行这些任务，通过定义对象及其行为(方法)，可以让对象来为我们完成任务，而不需要我们亲自去执行每一个细节。这种方式就像使用洗衣机洗衣服，我们只需设定好洗衣机的工作模式和放入衣物，洗衣机就会自动完成洗涤、漂洗、甩干等一系列操作，我们无需亲自动手，从而实现了"偷懒,找别人帮我做"的效果。

## 作用

主要针对大型软件设计而提出，可以使软件设计更加灵活，并且更好的进行代码复用。

## 三大特性

封装、继承、多态。

### **封装（Encapsulation）**

将数据（属性）和操作数据的方法（函数）捆绑在类中，对外隐藏实现细节。

**优势**​：

- 提高安全性（防止外部直接修改内部数据）
- 降低耦合度（内部修改不影响外部调用）
- 增强可维护性（代码模块化）

### **继承（Inheritance）**

子类继承父类的属性和方法，并可扩展或重写父类功能。

**核心价值**​：

- 避免重复代码（DRY原则）
- 支持层次化设计（如动物→猫/狗）
- 简化代码维护（共性逻辑集中管理）

### **多态（Polymorphism）**

同一方法在不同类中有不同实现，通过接口或继承实现。

**作用**​：

- 提高扩展性（新增子类无需修改调用代码）
- 增强灵活性（如`动物.叫()`，猫、狗表现不同）
- 支持面向接口编程（依赖抽象而非具体实现）

# 类与对象

## 含义

### 类

对一系列具有相同属性和行为的事物的统称。

- 属性：即特征(如人的身高和体重等)，用来描述是什么样子的。

- 行为：即所具备的功能或能够执行的动作(如人会走路、会说话等)，用来说明能够做什么。

注意：**类是一个抽象的概念，并不是真存在的事物**，可以把它理解成一张洗衣机的设计图，根据这张设计图能够设计出很多的洗衣机，也就是我们所说的对象。

### 对象

类的具体表现，是类创建出来的真实存在的事物，也是面向对象编程思想的核心。

注意：Python 中万物皆为对象。数据、函数甚至类等都是对象，这说明 Python 天生就是面向对象的。

```python
import copy

def func():
    print("__func__")

print(type("Hello"))  # <class 'str'>
print(type(3.14))  # <class 'float'>
print(type(True))  # <class 'bool'>
print(type([]))  # <class 'list'>
print(type({}))  # <class 'dict'>
print(type(set()))  # <class 'set'>
print(type(func))  # <class 'function'>
print(type(copy))  # <class 'module'>
```

> 注意：面相对象编程中，现有类，再有对象。

## 定义类

语法形式：

```python
class ClassName():
  '''类的描述信息'''
  类体
```

注意：类名应当严格遵守Python的标识符命名规则，并目采用大驼峰命名法，以及命名应具有描述性和直观性，即见名知意。

代码示例：

```python
class Person:
    """人类"""
    pass  # 占位符，表示空语句，当语法上需要语句但又不想执行任何操作时使用
```

定义完类之后，并不会立即产生一个具体的实例(对象)。正如前面所言，类就像是洗衣机的设计图，它详细说明了洗衣机应有的外观和功能，但设计图本身并不具备洗衣机的实际功能。要创建一个或多个具有这些功能的洗衣机(即类的实例)，我们需要根据这个设计图来实例化对象。

## 实例化对象

语法格式如下：

```python
对象名 = 类名()
```

代码示例：
```python
# 定义类
class Person:
    """人类"""
    pass


# 实例化对象
p1 = Person()
print(p1)  # <__main__.Person object at 0x104aaa120>
```

注意：在Python中，一个已定义的类能够作为模板，用于创建多个独立且结构相同的对象实例，正如一张设计图纸能够指导生产出多台洗衣机。

```python
p1 = Person()
print(p1)  # <__main__.Person object at 0x1057cf5c0>
p2 = Person()
print(p2)  # <__main__.Person object at 0x1057cf5f0>
p3 = Person()
print(p3)  # <__main__.Person object at 0x1057cf650>
```

内存地址不同，意味着创建的是彼此独立且不同的对象实例。

每当使用类来创建新的对象时，Python 都会在内存中为该对象分配一个新的空间。由于每个对象都在内存中占据不同的位置(即具有不同的内存地址)，因此这些对象实例是彼此独立且不同的。

## 方法

实例方法是类定义的成员函数，需由对象(实例)调用。这些方法至少包含一个 self 参数，该参数自动绑定到调用它的对象上，以便于在方法体内访问和修改对象的属性或状态。

```python
# 定义类
class Person:
    def study(self):  # 实例方法
        print("爱学习")
        print("self:", self)


p1 = Person()
p1.study()

"""
爱学习
self: <__main__.Person object at 0x105397590>
"""
```

在 Python 中，当对象调用其实例方法时，Python 解释器会自动将该对象的引用（即该对象在内存中的地址或标识符）作为第一个参数传递给实例方法的 `self` 参数。

> **注意**：Python 中将类中代表对象本身的参数命名为 `self` 是一种广泛遵循的约定，这样做有助于提高代码的可读性和一致性。然而，这并不是 Python 语言的硬性规定，开发者可以根据需要选择其他名称来替代 `se1f`，但这样做通常不推荐，因为它可能会降低代码的可读性和可维护性。

## 属性

在 Python 类中，根据属性的定义位置，可以将其明确区分为类属性和实例属性。

### 类属性

定义在类中，并且在方法体外的变量。它们与类本身紧密相关，而不是与类的任何单一实例(对象)相关联。因此，类属性天然具有共享性，能够在类的所有实例之间保持同一个值，允许所有通过该类创建的对象访问并(谨慎地)修改这个共享的值。

```python
class Person:
    age = 18  # 类属性（共性）

    def study(self):
        print("爱学习")


print(Person.age)  # 18

p1 = Person()
p2 = Person()

print(p1.age)  # 18
print(p2.age)  # 18
```

### 实例属性

在类的实例方法中通过 `self` 赋值的变量，它们仅作用于定义它们的对象(实例)上，即每个对象(实例)都拥有自己独立的实例属性副本。

```python
class Person:
    age = 18  # 类属性（共性）

    def study(self):
        print(f"{self.name}爱学习")  # self.name 实例属性（个性）


# 创建对象
p1 = Person()
# 给实例属性赋值
p1.name = "张三"
# 通过对象调用实例方法
p1.study()
```

## 构造函数 `__init__`

在Python中，`__init__` 方法是一个特殊的方法(也称为魔术方法或双下划线方法)，它是类的构造函数。当你创建一个类的新实例(对象)时，Python 会自动调用这个方法。

```python
class Person:
    # 构造函数
    def __init__(self):
        print("__init__")


# 实例化对象
p1 = Person()
p2 = Person()

"""
__init__
__init__
"""
```

`__init__` 方法的主要目的是初始化新创建的对象的状态，即给对象的实例属性设置初始值。

```python
class Person:
    # 类属性（共性）
    age = 18

    # 构造函数
    def __init__(self, name):
        # 实例属性
        self.name = name

    # 方法
    def study(self):
        print(f"{self.name}爱学习")


# 实例化对象
p1 = Person("张三")
p2 = Person("李四")

p1.study()  # 张三爱学习
p2.study()  # 李四爱学习
```

## 析构函数 `__del__`

在Python中，`__del__` 方法是一个特殊的方法(也称为魔术方法或双下划线方法)，它是类的析构函数。它在对象被销毁之前可能被 Python 的垃圾回收机制调用(如果对象不再被需要且垃圾回收器决定回收其内存)。

```python
class Person:
    def __del__(self):  # 析构函数
        print("__del__")


# 实例化对象
p1 = Person()
print(123)
print(456)

"""
123
456
__del__
"""

```

当使用 `del` 语句显式地删除对象时，Python 的垃圾回收机制在后续某个时间点可能会识别到该对象不再被需要，从而调用其 `__del__` 方法(如果定义了的话)。

```python
class Person:
    def __del__(self):  # 析构函数
        print("__del__")


# 实例化对象
p1 = Person()
print(123)
del p1
print(456)

"""
123
__del__
456
"""
```

> 注意：在Python中，`__del__` 方法的作用相对有限，因为 Python 拥有一套高效的自动垃圾回收机制。这是 Python 的一个显著优势，意味着当 Python 文件执行完毕或程序被明确终止时，Python 会自动回收并清理其占用的内存空间，有效防止了因内存泄漏而可能导致的系统资源耗尽问题。为了更直观地理解这一点，你可以尝试编写一个包含死循环的 Python 脚本并运行它，此时通过任务管理器可以看到 Python 进程正在占用CPU资源。然而，一旦你终止了这个脚本的执行，Python 进程及其所占用的资源会立即被操作系统回收，不会出现资源残留的情况，这是许多其他编程语言在资源管理上所不具备的特性。因此，在大多数情况下，我们不需要依赖  `__del__` 方法进行显式的资源清理，而是可以信任Python的垃圾回收机制来自动处理这些任务。

## 打印函数 `__str__`

在 Python 中，`__str__` 方法是一个特殊的方法(也称为魔术方法或双下划线方法)，用于定义当对象被转换为字符串时的行为。当你尝试输出对象(使用 `print` 函数)或在需要字符串表示(例如，在格式化字符串中)使用该对象时，Python 解释器会自动调用该对象的 `__str__` 方法，并使用其返回值作为对象的字符串表示。

```python
class A:
    pass


a = A()
print(a)  # <__main__.A object at 0x104ea20c0>


class B:
    def __str__(self):
        return "__str__方法的返回值"


b = B()
print(b)  # __str__方法的返回值
```

通过定义 `__str__` 方法，可以确保输出或记录对象时，输出的是易于理解的信息，而不是默认的、可能不太有用的类型名称和内存地址。这有助于提高代码的可读性和可维护性。

```python
class Person:
    def __init__(self, name, age, sex):  # 构造函数
        # 实例属性
        self.name = name
        self.age = age
        self.sex = sex

    def __str__(self):
        return f"Person(name={self.name}, age={self.age}, sex={self.sex})"


p1 = Person("张三", 32, "男")
p2 = Person("李四", 22, "女")
print(p1)  # Person(name=张三, age=32, sex=男)
print(p2)  # Person(name=李四, age=22, sex=女)
```

# 封装 & 单继承

## 封装

**封装** 是指将数据（属性）和操作数据的方法（行为）捆绑在一个类中，对外隐藏内部实现细节，仅暴露必要的接口，以提高代码的安全性、模块化和可维护性

类本质上就是一种封装机制，将属性和方法封装在内部，而实例化对象则是通过调用这些封装好的成员(属性和方法)来执行具体功能。

```python
class Person:
    """人类"""

    def __init__(self, name, age): # 构造函数
        self.name = name           # 实例属性
        self.age = age

    def say_hello(self):          # 实例方法
        print(f"你好呀，我是{self.name}，今年{self.age}岁")


# 实例化对象
p1 = Person("张三", 18)
# 通过对象调用实例方法
p1.say_hello()
# 修改实例属性
p1.age = 20
```

> **注意**：在Python中，没有内置的私有修饰符用于直接声明属性或方法为私有。如果希望属性或方法不被类外部直接访问，可以在其名称前添加双下划线（`__`）前缀。

```python
class Person:
    """人类"""

    def __init__(self, name, age):  # 构造函数
        self.name = name            # 实例属性（公共属性）
        self.__age = age            # 私有属性

    def say_hello(self):            # 实例方法
        print(f"你好呀，我是{self.name}，今年{self.age}岁")


# 实例化对象
p1 = Person("张三", 18)
# 尝试访问私有属性
print(p1.__age)  # AttributeError: 'Person' object has no attribute '__age'
```

在这个例子中，`__age` 是一个私有属性，因此它不能直接从类的外部被访问。

如果确实需要访问或修改 Python 中的私有属性（尽管这通常不被推荐，因为它破坏了封装原则），可以通过以下几种方式来实现:

1. **类中定义公开方法**：这是访问私有属性的推荐方式

   ```python
   class Person:
       """人类"""
   
       def __init__(self, name, age):  # 构造函数
           self.name = name            # 实例属性（公共属性）
           self.__age = age            # 私有属性
   
       def say_hello(self):            # 实例方法
           print(f"你好呀，我是{self.name}，今年{self.age}岁")
   
       # 定义公开方法访问私有属性
       def get_age(self):
           return self.__age
   
       # 定义公开方法修改私有属性
       def set_age(self, age):
           self.__age = age
   
   
   p1 = Person("张三", 18)
   print(p1.get_age())  # 18
   p1.set_age(20)
   print(p1.get_age())  # 20
   ```

2. **使用 Python 的内部机制（名称改写）**：在 Python 中，使用双下划线（`__`）开头的属性名实际上是一种命名约定，旨在表明该属性是私有的，不应在类外部直接访问。Python 解释器通过名称改写机制自动将这些属性名转换为 **\_类名\__属性名** 的形式。

   ```python
   # 通过名称改写机制访问私有属性：_类名__属性名（不推荐）
   print(p1._Person__age) 

> **注意**：尽管技术上可以通过 Python 的内部机制(名称改写)来访问私有属性(如 `p1._Person__age`)，但这种做法违背了封装原则，并且如果类的实现发生变化(例如类名改变)，代码将需要相应地进行更新，降低了代码的可维护性和可读性。因此，在实际开发中应避免这样做。

> **总结**：Python 并没有实现传统意义上的私有化，但它采用了一种命名约定，即在属性或方法名前添加双下划线（`__`）前缀，作为对内部成员私有性的标识，从而指导开发人员处理需要隐藏或保护的数据和方法。

## 继承

含义：在日常生活中，继承常指子女承接父辈的财产，体现了财富与责任的代际传递。而在 Python 编程中继承则是一种构建类之间关系的强大机制，它让类与类之间形成类似父子的层级结构，其中子类默认会"**继承**"父类的所有 **非私有属性和方法**。

作用：在编写类时，我们不必每次都从零开始。当发现新类与已存在的类之间存在合理的继承关系时，利用继承机制可以自然地实现代码的重用，避免重复编写相同的代码段，从而显著提升开发效率。

语法格式：

```python
class 类名(父类名):
  '''类的描述信息'''
  类体
```

> **注意**：可以指定多个父类，父类名之间用逗号 `,` 分隔，若未指定父类，则默认继承自 `object` 类（Python的顶级父类，又称基类或根类）。

在 Python 中，根据子类继承的父类数量，我们可以将继承分成 **单继承** 和 **多继承**。单继承指的是子类只从一个父类继承属性和方法，而多继承则指的是子类可以从多个父类继承属性和方法。

### 单继承

在 Python中，单继承是指一个子类继承一个父类的所有非私有属性和方法。这是面向对象编程中最基本的继承形式，它简化了类的层次结构，使得类的关系更加清晰，同时也促进了代码的重用。

示例：

```python
class Person:
    kind = "人类"  # 类属性

    def __init__(self, name):
        self.name = name  # 实例属性

    def greet(self):
        print(f"你好，我是{self.name}，我属于{self.kind}。")


# 定义子类，继承自 Person 类
class Student(Person):
    pass


# 通过类访问类属性
print(Student.kind)
# 实例化子类对象
stu = Student("张三")
# 通过对象访问实例属性
print(stu.name)
# 通过对象调用实例方法
stu.greet()
```

### 继承的传递性

Python 的继承具有传递性，这意味着如果一个类A继承自类B，而类B又继承自类C，那么类A能够间接地继承类C及其所有祖先类中定义的非私有属性和方法。这确保了子类能够继承并使用其所有父类及更上层父类的非私有成员。

示例：

```python
class C:
    c_var = "C类类属性"

    def c_method(self):
        print("C类实例方法")


class B(C):
    b_var = "B类类属性"

    def b_method(self):
        print("B类实例方法")


class A(B):
    a_var = "A类类属性"

    def a_method(self):
        print("A类实例方法")


# 通过A类访问类属性
print(A.a_var)  # 输出: A类类属性
print(A.b_var)  # 输出: B类类属性
print(A.c_var)  # 输出: C类类属性


# 实例化A类对象
a = A()
a.a_method()  # 输出: A类实例方法
a.b_method()  # 输出: B类实例方法
a.c_method()  # 输出: C类实例方法
```

## 方法的重写

在 Python 中，方法的重写是指子类定义了一个与父类中同名的方法。分类:

1. **覆盖父类方法**

   当子类定义了一个与父类同名的法时，子类的方法就会覆盖父类中的方法。

   ```python
   class Parent:
       def show(self):
           print("Parent class")
   
   
   class Child(Parent):
       def show(self):
           print("Child class")
   
   
   child = Child()
   child.show()  # Child class
   ```

2. **对父类方法进行扩展**

   通过在子类中定义一个与父类同名的方法，并在该方法内部首先调用父类的同名方法（以保留父类的行为），然后添加额外的逻辑来实现。

   ```python
   class Parent:
       def show(self):
           print("Parent class")
   
   
   class Child(Parent):
       def show(self):
           # 调用父类的同名方法
           Parent.show(self)
           super(Child, self).show()
           super().show()  # Python 3.x+ 推荐
           print("Child class")
   
   
   child = Child()
   child.show()
   
   """
   Parent class
   Parent class
   Parent class
   Child class
   """
   ```

## 新式类写法

在 Python中，新式类是指任何直接或间接继承自 `object` 的类，这涵盖了直接继承 `obyect` 的类以及它们所有的子类，无论这些子类在继承链上的深度如何。然而，在 Python 3.x 版本中，这一概念得到了极大的简化：**所有的类默认都是新式类**，即当你定义一个类时，它会自动继承自 `object`，从而无需像 Python 2.x 中那样显式地从 `object` 继承。这一变化不仅简化了类的定义，还使得 Python 3.x 的面向对象编程更加直观、灵活和方便。

在 Python 3.x 版本中，定义类有以下三种方式：

1. 写法一：推荐

   ```python
   class 类名:
     '''类的描述信息'''
     类体
   ```

2. 写法二：

   ```python
   class 类名():
     '''类的描述信息'''
     类体
   ```

   这种方式提供了不必要的括号且可能引起混淆，因此不推荐使用。

3. 写法三：

   ```python
   class 类名(object):
     '''类的描述信息'''
     类体
   ```

   尽管在 Python 3.x 中不是必需的，但这种写法保持了与Python 2.x 代码的兼容性，对于那些需要同时支持两个版本 Python 的开发者来说，或者团队中有成员偏好于这种写法的情况，使用这种写法是合理的。

在 Python 3.x 版本中，这三种定义类的方式在功能上没有区别，因为所有的类默认都是新式类，即它们都继承自 `object`。不过，从最佳实践和代码清晰度的角度来看，主要倾向于第一种或第三种方式。

总结：在 Python3.x 版本中，`object` 是所有类的顶级父类。如果一个类在定义时没有显式地指定任何父类，它也会自动继承自 `object`，因此 Python 3.x 版本中的所有类默认都是新式类。

# 多继承 & 多态

## 多继承

单继承指的是子类仅直接继承自一个父类，从而获取该父类的属性和方法。而多继承则允许子类同时继承自多个父类，从而能够获取并访问这些父类的所有非私有属性和方法。

语法格式：

```python
class Child(ParentA, ParentB):
  pass
```

```python
class ParentA:
    a_var = "ParentA"

    def a_method(self):
        return "ParentA method"


class ParentB:
    b_var = "ParentB"

    def b_method(self):
        return "ParentB method"


class Child(ParentA, ParentB):
    pass


# 通过子类访问类属性
print(Child.a_var)  # 输出: ParentA
print(Child.b_var)  # 输出: ParentB

# 实例化子类对象
child = Child()
# 通过子类对象访问父类方法
child.a_method()  # 输出: ParentA method
child.b_method()  # 输出: ParentB method
```

思考：当不同的父类中存在同名的方法，子类对象在调用方法时，会调用哪一个父类中的方法呢?

### 不同父类间存在同名方法

在Python中，当不同父类存在同名方法时，其调用顺序由**方法解析顺序（MRO）**决定，具体规则如下：

1. **MRO规则**

   Python使用**C3线性化算法**计算MRO，遵循以下原则：

   - **子类优先**：子类的方法优先于父类被调用。
   - **声明顺序保留**：继承列表中靠左的父类优先级更高（从左到右）。
   - **调性**：父类的MRO顺序在子类中保持一致

2. **调用顺序示例**

   ```python
   class A:
       def method(self): print("A")
   class B(A):
       def method(self): print("B")
   class C(A):
       def method(self): print("C")
   class D(B, C): pass
   
   d = D()
   d.method()  # 输出: B（按MRO顺序 D -> B -> C -> A）
   ```

   - 若`D`的继承顺序改为`class D(C, B)`，则输出`C`

3. **关键方法**

   - 查看MRO：通过`类名.__mro__`或`类名.mro()`
   - `super()`函数按MRO顺序调用下一个父类方法

4. **菱形继承（钻石问题）**

   - 若多个父类继承自同一祖先（如`B`和`C`继承`A`），Python确保祖先方法仅调用一次

**总结**：在多继承中，当存在同名方法时，Python 将严格依据 `__mro__` 定义的顺序，从左至右遍历类及其所有基类以搜索方法。如果在当前类或任一父类中找到了对应的方法，Python 就会立即执行该方法，并停止进一步的搜索。如果遍历完整个 `__mro__` 元组，包括最基础的 `object` 类，都没有找到对应的方法，那么Python 会抛出一个 `AttributeError` 异常，明确指出该方法未被定义。

> 注意：开发时，应尽量避免这种容易产生混淆的情况。也就是说当不同父类之间存在同名的属性或方法时，应该尽量避免使用多继承。

> 弊端：多继承虽然提供了灵活性，但也容易引发冲突，特别是当多个父类之间存在同名方法时，这会增加代码设计的复杂性和维护难度。因此，在设计类继承结构时，应谨慎使用多继承。

## 多态

多态是指同一种行为在不同类型的对象上展现出多样化的表现形式。以动物为例，小猫、小狗和小羊虽都具备"叫"的行为，但各自的具体表现(叫声)却截然不同。同样地，在"飞"这一行为上，鸟类与飞机虽然都实现了飞翔，但它们的飞行方式、机制和效果则大相径庭。多态性正是对这种"同一行为，多样表现"特性的精准描述，它使得软件设计更加灵活，能够更好地适应不同类型对象的需求和行为差异。

前提：

1. 继承
2. 方法的重写

示例：

```python
class Animal:
    """动物类"""
    def speak(self):
        print("动物在叫")


class Dog(Animal):
    """狗类"""
    def speak(self):
        print("汪汪汪")


class Cat(Animal):
    """猫类"""
    def speak(self):
        print("喵喵喵")


# 定义一个统一的接口，接收一个 Animal 类型的参数（或其子类对象），并调用该对象的 speak 方法
def animal_speak(animal: Animal):
    """统一的接口，用于调用传入对象的 speak 方法"""
    animal.speak()


# 实例化对象
dog = Dog()
cat = Cat()
# 调用接口，传入不同的对象，产生不同的执行结果
# 由于多态性，animal_speak 能够识别传入对象的实际类型，并调用相应的 speak 方法
animal_speak(dog)  # animal = dog ==> animal.speak() = dog.speak() ==> 输出：汪汪汪
animal_speak(cat)  # animal = cat ==> animal.speak() = cat.speak() ==> 输出：喵喵喵
```

需求：利用多态的思想，编写程序模拟"主人"（`Master`）喂养"宠物"（`Pet`）的场景。

```python
# 定义宠物类
class Pet:
    """宠物类"""
    def eat(self):
        print("宠物在吃东西...")


# 定义主人类
class Master:
    """主人类"""
    def feed(self, pet):
        print("主人在喂养宠物...")
        pet.eat()  # 调用宠物的吃东西方法
        print("主人喂养宠物完成！")


# 定义狗类，继承自 Pet
class Dog(Pet):
    """狗类"""
    def eat(self):
        print("狗在吃骨头...")


# 定义猫类，继承自 Pet
class Cat(Pet):
    """猫类"""
    def eat(self):
        print("猫在吃鱼...")


# 定义金鱼类，继承自 Pet
class Goldfish(Pet):
    """金鱼类"""
    def eat(self):
        print("金鱼在吃鱼食...")


# 实例化对象
ma = Master()
dog = Dog()
cat = Cat()
goldfish = Goldfish()

# 主人喂养不同的宠物
ma.feed(dog)  # 喂养狗

```

## 静态方法

Python中的**静态方法**是通过`@staticmethod`装饰器定义的类方法，它**不依赖类实例或类本身**，主要用于实现与类逻辑相关但无需访问实例或类属性的功能。以下是核心要点：

定义格式：

```python
@staticmethod
def 方法名(形参):
  方法体
```

代码示例：

```python
class MathUtils:
    @staticmethod
    def add(a, b):
        return a + b


print(MathUtils.add(3, 5))  # 输出: 8
```

主要特点：

- **无状态访问**：不能直接访问实例属性（`self`）或类属性（`cls`）。
- **共享性**：所有实例共享同一份静态方法。
- **调用方式**：推荐通过类名调用（如`ClassName.method()`），实例调用也可行但不推荐

应用场景：

- **工具函数**（如数学计算、数据验证）
- **工厂方法**（创建类实例）
- **辅助功能**（如日期处理、常量管理）

注意事项：

- 避免滥用，仅用于逻辑相关但无状态依赖的场景
- 多继承时，同名静态方法按父类声明顺序调用

静态方法通过解耦状态依赖提升代码模块化，常见于工具类和工厂模式中

## 类方法

Python中的 **类方法** 是通过 `@classmethod` 装饰器定义的、绑定到类而非实例的方法。

定义格式：

```python
@classmethod
def 方法名(cls, 形参):
  方法体
```

类方法一般和类属性配合使用，当方法中需要使用类(如访问类属性，调用其他类方法或实例化对象)时可以考虑将该方法定义为类方法。

核心特性：

1. **参数与访问权限**
   - 第一个参数为`cls`（指向类本身），可访问和修改**类属性**，但无法直接操作实例属性
   - 对比其他方法：
     - **实例方法**：参数为 `self`，操作实例属性。
     - **静态方法**：无 `self` 或 `cls`，仅作工具函数
2. **调用方式**
   - 可通过**类名直接调用**（如`MyClass.method()`）或**实例调用**（不推荐）

主要用途：

1. **工厂方法**

   提供替代构造器，支持灵活实例化（如从字典/字符串创建对象）

   ```python
   class Person:
       @classmethod
       def from_dict(cls, data):
           return cls(data['name'], data['age'])
   ```

2. **操作类状态**

   修改类变量（如计数器、全局配置）

   ```python
   class Car:
       wheels = 4
       @classmethod
       def set_wheels(cls, num):
           cls.wheels = num
   ```

3. **继承友好**

   子类调用时，`cls`自动指向当前类，便于多态设计

# 抽象基类 - `abc`

Python 中的 `abc` 模块（**Abstract Base Classes**，抽象基类）是标准库中用于定义抽象类和接口的工具，通过强制子类实现特定方法来实现多态性和代码规范。以下是核心要点：

## 核心功能

**定义抽象基类**

使用 `abc.ABC` 或元类 `abc.ABCMeta` 创建抽象类，例如：

```python
from abc import ABC, abstractmethod

class Shape(ABC):
    @abstractmethod
    def area(self):
        pass
```

子类必须实现 `area()`，否则无法实例化

**抽象方法标记**

用 `@abstractmethod` 装饰器声明必须实现的方法。若子类未实现，抛出 `TypeErro`

**虚拟子类机制**

通过 `register()` 或 `__subclasshook__` 将非继承类注册为抽象基类的“虚拟子类”，例如：

```python
MyABC.register(tuple)  # tuple 被视为 MyABC 的子类[1,6](@ref)
```

## 主要组件

| 组件                | 作用                                                         |
| :------------------ | :----------------------------------------------------------- |
| `abc.ABC`           | 辅助类，通过继承方式定义抽象基类（避免直接使用元类）         |
| `abc.ABCMeta`       | 元类，用于动态创建抽象基类                                   |
| `@abstractmethod`   | 标记抽象方法                                                 |
| `@abstractproperty` | 标记抽象属性（Python 3.3+ 推荐用 `@property` + `@abstractmethod`） |

## 使用场景

1. **接口规范**：确保子类实现特定方法（如数据处理接口 `load_data()`/`save_data()`）
2. **类型检查**：通过 `issubclass()` 或 `isinstance()` 验证类是否满足接口
3. **插件系统**：强制插件实现统一接口

## 示例代码

```python
from abc import ABC, abstractmethod

class Animal(ABC):
    @abstractmethod
    def sound(self): pass

class Dog(Animal):
    def sound(self):
        return "Bark"

# 错误示例：未实现抽象方法
class Cat(Animal): pass  # TypeError: Can't instantiate abstract class Cat
```
