# 准备工作
```postgresql
-- 删除已有表
DROP TABLE IF EXISTS company;
-- 新建表
CREATE TABLE company (
   id INT PRIMARY KEY     NOT NULL,
   name           TEXT    NOT NULL,
   age            INT     NOT NULL,
   address        CHAR(50),
   salary         REAL
);
```

# 插入数据 INSERT INTO

语法格式：

```postgresql
INSERT INTO <table_name> (column1, column2, ...columnN) 
VALUES 
	(value1, value2, ...valueN),
	(value1, value2, ...valueN),
	...
```

在使用 INSERT INTO 语句时，字段列必须和数据值数量相同，且顺序也要对应。

如果我们向表中的所有字段插入值，则可以不需要指定字段，只需要指定插入的值即可：

```postgresql
INSERT INTO <table_name>  VALUES (value1,value2,value3,...valueN);
```

**实例**

在 COMPANY 表中插入以下数据：

```postgresql
INSERT INTO COMPANY (id,name,age,address,salary) 
VALUES 
	(1, '张三', 32, '成都', 20000.00);
```

以下插入语句忽略 salary 字段：

```shell
INSERT INTO COMPANY (id,name,age,address) 
VALUES 
	(2, '李四', 32, '自贡');
```

以下实例插入多行：

```postgresql
INSERT INTO COMPANY (id,name,age,address,salary) 
VALUES 
  (3, '赵二', 31, '乐山', 20000.00),
  (4, '王五', 33, '绵阳', 65000.00),
  (5, '刘七', 33, '宜宾', 85000.00);
```

使用 SELECT 语句查询表格数据：

```postgresql
postgres=# SELECT * FROM company;
 id | name | age | address | salary 
----+------+-----+---------+--------
  1 | 张三 |  32 | 成都    |  20000
  2 | 李四 |  32 | 自贡    |

  4 | 王五 |  33 | 绵阳    |  65000
  5 | 刘七 |  33 | 宜宾    |  85000
(5 行记录)
```

# 更新数据 UPDATE

@See https://www.runoob.com/postgresql/postgresql-update.html

使用 UPDATE 语句来更新 PostgreSQL 表中的数据。

语法格式：

```postgresql
UPDATE <table_name>
SET column1 = value1, column2 = value2...., columnN = valueN
WHERE [condition];
```

- 可以同时更新一个或者多个字段
- 可以在 WHERE 子句中指定任何条件。

**实例**

1）将 id 为 2 的那条数据中的 salary 字段设置为 15000

```postgresql
postgres=# UPDATE company SET salary = 15000 WHERE id = 2;
UPDATE 1
```

得到结果如下：

```postgresql
 id | name | age | address | salary 
----+------+-----+---------+--------
  1 | 张三 |  32 | 成都    |  20000
  3 | 赵二 |  31 | 乐山    |  20000
  4 | 王五 |  33 | 绵阳    |  65000
  5 | 刘七 |  33 | 宜宾    |  85000
  2 | 李四 |  32 | 自贡    |  15000
```

2）同时更新 salary 字段和 address 字段的值：

```postgresql
postgres=# UPDATE company SET address = '未知', salary = 20000;
UPDATE 5
```

得到结果如下：

```postgresql
 id | name | age | address | salary 
----+------+-----+---------+--------
  1 | 张三 |  32 | 未知    |  20000
  3 | 赵二 |  31 | 未知    |  20000
  4 | 王五 |  33 | 未知    |  20000
  5 | 刘七 |  33 | 未知    |  20000
  2 | 李四 |  32 | 未知    |  20000
```

# 删除数据 DELETE

@See https://www.runoob.com/postgresql/postgresql-delete.html

使用 DELETE 语句来删除 PostgreSQL 表中的数据。

语法格式：

```postgresql
DELETE FROM <table_name> WHERE <condition>;
```

如果没有指定 WHERE 子句，PostgreSQL 表中的所有记录将被删除。

一般我们需要在 WHERE 子句中指定条件来删除对应的记录，条件语句可以使用 AND 或 OR 运算符来指定一个或多个。

**实例**

1）删除 id = 2 的记录：

```postgresql
postgres=# DELETE FROM company WHERE id = 2;
DELETE 1
```

2）删除所有记录：

```postgresql
postgres=# DELETE FROM company;
DELETE 4
```

## 运算符

@See https://www.runoob.com/postgresql/postgresql-operators.html

运算符是一种告诉编译器执行特定的数学或逻辑操作的符号。

PostgreSQL 运算符是一个保留关键字或字符，一般用在 WHERE 语句中，作为过滤条件。

常见的运算符有：

- 算术运算符：`+` `-` `*` `/` `%` `!` ...
- 比较运算符：`>` `>=` `<` `<=` `=` `!=` ...
- 逻辑运算符：`AND` `NOT` `OR`
- 按位运算符：`&` `|` `^`

**实例**

1）**查询 name 和 age，然后 age 在原有基础上加 100**：

```postgresql
postgres=# SELECT name,age+100 FROM company;
 name | ?column? 
------+----------
 赵云 |      128
 吕布 |      130
 貂蝉 |      122
 李白 |      126
 韩信 |      124
 妲己 |      120
 孙策 |      125
 张良 |      135
 大乔 |      123
(9 行记录)
```

可以看到， age 列的值是没有问题的，但是列标题是 `?column?`，此时我们可以通过 `AS` 关键字给列标题赋值，如：

```postgresql
postgres=# SELECT name,age+100 AS new_age  FROM company;
 name | new_age 
------+---------
 赵云 |     128
 吕布 |     130
 貂蝉 |     122
 李白 |     126
 韩信 |     124
 妲己 |     120
 孙策 |     125
 张良 |     135
 大乔 |     123
(9 行记录)
```

2）**查询 age >= 25，同时 salary > 30000 的记录**：

```postgresql
postgres=# SELECT * FROM company WHERE age >= 25 AND salary > 30000;
 id | name | age | address | salary 
----+------+-----+---------+--------
  4 | 李白 |  26 | 碎叶    |  35000
  7 | 孙策 |  25 | 江东    |  32000
  8 | 张良 |  35 | 颍川    |  40000
(3 行记录)
```

## 表达式

@See https://www.runoob.com/postgresql/postgresql-expressions.html

表达式是由一个或多个的值、运算符、PostgresSQL 函数组成的。

PostgreSQL 表达式类似一个公式，我们可以将其应用在查询语句中，用来查找数据库中指定条件的结果集。

SELECT 语句的语法格式如下：

```postgresql
SELECT column1, column2, columnN
FROM <table_name>
WHERE [CONDITION | EXPRESSION];
```

PostgreSQL 的表达式可以有不同类型：

- 布尔表达式
- 数字表达式
- 日期表达式

### 布尔表达式

布尔表达式是根据一个指定条件来读取数据：

```postgresql
SELECT column1, column2, columnN
FROM <table_name>
WHERE SINGLE VALUE MATCHTING EXPRESSION;
```

以下使用了布尔表达式（**SALARY=25000**）来查询数据：

```postgresql
postgres=# SELECT * FROM company WHERE salary = 25000;
 id | name | age | address | salary 
----+------+-----+---------+--------
  1 | 赵云 |  28 | 常山    |  25000
(1 行记录)
```

### 数字表达式

数字表达式常用于查询语句中的数学运算：

```postgresql
SELECT numerical_expression as  OPERATION_NAME
[FROM table_name WHERE CONDITION] ;
```

**numerical_expression** 是一个数学运算表达式，实例如下：

```postgresql
postgres=# SELECT (10 + 10) AS sum_result;
 sum_result 
------------
         20
(1 行记录)
```

此外 PostgreSQL 还内置了一些数学函数，如：

- `AVG()` ： 返回一个表达式的平均值
- `SUM()` ： 返回指定字段的总和
- `COUNT()` ： 返回查询的记录总数

1）以下实例查询 COMPANY 表的记录总数：

```postgresql
postgres=# SELECT COUNT(*) AS total_count FROM company;
 total_count 
-------------
           9
(1 行记录)
```

### 日期表达式

日期表达式返回当前系统的日期和时间，可用于各种数据操作，以下实例查询当前时间：

```postgresql
postgres=# SELECT CURRENT_TIMESTAMP;
       current_timestamp       
-------------------------------
 2025-06-24 16:13:44.276889+08
(1 行记录)
```

## WHERE

@See https://www.runoob.com/postgresql/postgresql-where-clause.html

在 PostgreSQL 中，当我们需要根据指定条件从单张表或者多张表中查询数据时，就可以在 SELECT 语句中添加 WHERE 子句，从而过滤掉我们不需要数据。

WHERE 子句不仅可以用于 SELECT 语句中，同时也可以用于 UPDATE，DELETE 等等语句中。

以下是 SELECT 语句中使用 WHERE 子句从数据库中读取数据的通用语法：

```postgresql
SELECT column1, column2, columnN
FROM <table_name>
WHERE [condition1]
```

我们可以在 WHERE 子句中使用比较运算符或逻辑运算符，例如 **>, <, =, LIKE, NOT** 等等。

以下几个实例我们使用逻辑运算符来读取表中的数据。

> **`AND`**

找出 **AGE(年龄)** 字段大于等于 25，并且 **SALARY(薪资)** 字段大于等于 35000 的数据：

```postgresql
----+------+-----+---------+--------
  4 | 李白 |  26 | 碎叶    |  35000
  8 | 张良 |  35 | 颍川    |  40000
(2 行记录)
```

找出 **AGE(年龄)** 字段大于等于 25，或者 **SALARY(薪资)** 字段大于等于 35000 的数据：

## AND & OR

@See https://www.runoob.com/postgresql/postgresql-and-or-clauses.html

在 PostgreSQL 中，AND 和 OR 也叫连接运算符，在查询数据时用于缩小查询范围，我们可以用 AND 或者 OR 指定一个或多个查询条件。

> **`AND`**

AND 运算符表示一个或者多个条件必须同时成立。

在 WHERE 子句中，AND 的使用语法如下：

```postgresql
SELECT column1, column2, columnN
FROM table_name
WHERE [condition1] AND [condition2]...AND [conditionN];
```

以下实例读取 AGE 字段大于 25 且 SALARY 字段大于等于 30000 的所有记录：

```postgresql
postgres=# SELECT * FROM company WHERE age > 25 AND salary >= 30000;
 id | name | age | address | salary 
----+------+-----+---------+--------
  2 | 吕布 |  30 | 洛阳    |  30000
  4 | 李白 |  26 | 碎叶    |  35000
  8 | 张良 |  35 | 颍川    |  40000
(3 行记录)
```

> **`OR`**

OR 运算符表示多个条件中只需满足其中任意一个即可。

在 WHERE 子句中，OR 的使用语法如下：

```postgresql
SELECT column1, column2, columnN
FROM table_name
WHERE [condition1] OR [condition2]...OR [conditionN]
```

以下实例读取 AGE 字段大于等于 25 或 SALARY 字段大于等于 30000 的所有记录：

```postgresql
postgres=# SELECT * FROM company WHERE age > 25 OR salary >= 30000;
```

## LIKE

在 PostgreSQL 数据库中，我们如果要获取包含某些字符的数据，可以使用 **LIKE** 子句。

在 LIKE 子句中，通常与通配符结合使用，通配符表示任意字符，在 PostgreSQL 中，主要有以下两种通配符：

- 百分号 **%**
- 下划线 **_**

如果没有使用以上两种通配符，LIKE 子句和等号 **=** 得到的结果是一样的。

以下是使用 LIKE 子句搭配百分号 **%** 和下划线 **_** 从数据库中获取数据的通用语法：

```postgresql
SELECT FROM <table_name> WHERE column LIKE 'XXXX%';
或者
SELECT FROM <table_name> WHERE column LIKE '%XXXX%';
或者
SELECT FROM <table_name> WHERE column LIKE 'XXXX_';
或者
SELECT FROM <table_name> WHERE column LIKE '_XXXX';
或者
SELECT FROM <table_name> WHERE column LIKE '_XXXX_';
```

你可以在 WHERE 子句中指定任何条件。

你可以使用 AND 或者 OR 指定一个或多个条件。

**XXXX** 可以是任何数字或者字符。

**实例**

下面是 LIKE 语句中演示了 **%** 和 **_** 的一些差别:

| 实例                             | 描述                                                   |
| :------------------------------- | :----------------------------------------------------- |
| WHERE SALARY::text LIKE '200%'   | 找出 SALARY 字段中以 200 开头的数据。                  |
| WHERE SALARY::text LIKE '%200%'  | 找出 SALARY 字段中含有 200 字符的数据。                |
| WHERE SALARY::text LIKE '_00%'   | 找出 SALARY 字段中在第二和第三个位置上有 00 的数据。   |
| WHERE SALARY::text LIKE '2\_%_%' | 找出 SALARY 字段中以 2 开头的字符长度大于 3 的数据。   |
| WHERE SALARY::text LIKE '%2'     | 找出 SALARY 字段中以 2 结尾的数据                      |
| WHERE SALARY::text LIKE '_2%3'   | 找出 SALARY 字段中 2 在第二个位置上并且以 3 结尾的数据 |
| WHERE SALARY::text LIKE '2___3'  | 找出 SALARY 字段中以 2 开头，3 结尾并且是 5 位数的数据 |

在 PostgreSQL 中，LIKE 子句是只能用于对字符进行比较，因此在上面例子中，我们要将整型数据类型转化为字符串数据类型。

下面实例将找出 AGE 以 3 开头的数据：

```postgresql
postgres=# SELECT * FROM company WHERE age::TEXT LIKE '3%';
 id | name | age | address | salary 
----+------+-----+---------+--------
  2 | 吕布 |  30 | 洛阳    |  30000
  8 | 张良 |  35 | 颍川    |  40000
(2 行记录)
```

下面实例将找出 ADDRESS 字段中含有 **阳** 字符的数据：

```postgresql
postgres=# SELECT * FROM company WHERE address LIKE '%阳%';
 id | name | age | address | salary 
----+------+-----+---------+--------
  2 | 吕布 |  30 | 洛阳    |  30000
```

## GROUP BY

@See https://www.runoob.com/postgresql/postgresql-group-by.html

在 PostgreSQL 中，**GROUP BY** 语句和 SELECT 语句一起使用，用来对相同的数据进行分组。

GROUP BY 在一个 SELECT 语句中，放在 WHERE 子句的后面，ORDER BY 子句的前面。

语法格式：

```postgresql
SELECT column-list
FROM table_name
WHERE [ conditions ]
GROUP BY column1, column2....columnN
ORDER BY column1, column2....columnN
```

GROUP BY 子句必须放在 WHERE 子句中的条件之后，必须放在 ORDER BY 子句之前。

在 GROUP BY 子句中，你可以对一列或者多列进行分组，但是被分组的列必须存在于列清单中。

## HAVING

@See https://www.runoob.com/postgresql/postgresql-having.html

HAVING 子句可以让我们筛选分组后的各组数据。

WHERE 子句在所选列上设置条件，而 HAVING 子句则在由 GROUP BY 子句创建的分组上设置条件。

下面是 HAVING 子句在 SELECT 查询中的位置：

```postgresql
SELECT
FROM
WHERE
GROUP BY
HAVING
ORDER BY
```

HAVING 子句必须放置于 GROUP BY 子句后面，ORDER BY 子句前面，下面是 HAVING 子句在 SELECT 语句中基础语法：

```postgresql
SELECT column1, column2
FROM table1, table2
WHERE [ conditions ]
GROUP BY column1, column2
HAVING [ conditions ]
ORDER BY column1, column2
```

**实例**

筛选高绩效部门（A级员工≥2人的部门）

```postgresql
SELECT department, COUNT(*) AS a_grade_count 
FROM emp_performance 
WHERE grade = 'A' 
GROUP BY department 
HAVING COUNT(*) >= 2;
```

结果示例：

```postgresql
 department | a_grade_count 
------------+---------------
 技术部     |             2
(1 行记录)
```

## WITH

@See https://www.runoob.com/postgresql/postgresql-with.html

PostgreSQL 中的 `WITH` 语句（也叫 **CTE，公共表表达式**）可以理解为一个 **临时表**，它能在一条复杂的查询中定义一个小查询，然后像普通表一样重复使用。通俗来说，就像写作文时先列个提纲，后续直接引用这个提纲，避免重复写同样的内容。

语法格式：

1）**基础语法**

```postgresql
WITH cte_name [(column1, column2, ...)] AS (
    -- 子查询（SELECT/INSERT/UPDATE/DELETE）
    SELECT ...
)
-- 主查询
SELECT * FROM cte_name;
```

- `cte_name`：临时表的名称，可在主查询中引用
- `(column1, column2, ...)`：可选，显式指定临时表的列名。若省略，列名由子查询决定
- **子查询**：可以是 `SELECT`、`INSERT`、`UPDATE` 或 `DELETE`（需搭配 `RETURNING` 子句）

示例：

```postgresql
WITH sales_summary AS (
    SELECT region, SUM(amount) AS total 
    FROM orders 
    GROUP BY region
)
SELECT * FROM sales_summary WHERE total > 10000;
```

2）**多重CTE**

可定义多个临时表，用逗号分隔：

```postgresql
WITH 
    cte1 AS (SELECT ...),
    cte2 AS (SELECT ...)
SELECT * FROM cte1 JOIN cte2 ON ...;
```

示例：

```postgresql
WITH 
    regional_sales AS (
        SELECT region, SUM(amount) AS total 
        FROM orders 
        GROUP BY region
    ),
    top_regions AS (
        SELECT region 
        FROM regional_sales 
        WHERE total > (SELECT AVG(total) FROM regional_sales)
    )
SELECT * FROM orders WHERE region IN (SELECT region FROM top_regions);
```

3）**递归语法（`RECURSIVE`）**

用于处理层次结构数据（如树形结构）：

```postgresql
WITH RECURSIVE cte_name AS (
    -- 非递归项（基础查询）
    SELECT ... FROM table WHERE ...
    UNION [ALL]
    -- 递归项（引用自身）
    SELECT ... FROM cte_name JOIN table ON ...
)
SELECT * FROM cte_name;
```

**说明**：

- **`UNION ALL`**：保留重复行，`UNION` 去重
- **终止条件**：递归项需包含终止条件（如 `WHERE n < 100`），否则会无限循环

**示例**（计算 1 到 100 的和）：

```postgresql
WITH RECURSIVE t(n) AS (
    VALUES (1)          -- 初始值
    UNION ALL
    SELECT n+1 FROM t WHERE n < 100  -- 递归终止条件
)
SELECT SUM(n) FROM t;
```

4）**数据修改语句**

`WITH` 可与 `INSERT`/`UPDATE`/`DELETE` 结合，通过 `RETURNING` 返回操作结果：

```postgresql
WITH moved_rows AS (
    DELETE FROM products 
    WHERE date < '2025-01-01'
    RETURNING *
)
INSERT INTO archive_products 
SELECT * FROM moved_rows;
```

> **注意**：数据修改语句需在顶层 `WITH` 中定义，且执行顺序不可预测

总结：

1. **作用域**：CTE 仅在当前查询中有效
2. **性能优化**：避免重复计算，但 PostgreSQL 12+ 支持 `NOT MATERIALIZED` 选项，允许外层条件推送至 CTE
3. **递归限制**：需确保递归查询能终止，可通过 `UNION` 去重或检查路径数组避免循环

## DISTINCT

@See https://www.runoob.com/postgresql/postgresql-distinct.html

在 PostgreSQL 中，DISTINCT 关键字与 SELECT 语句一起使用，用于去除重复记录，只获取唯一的记录。

我们平时在操作数据时，有可能出现一种情况，在一个表中有多个重复的记录，当提取这样的记录时，DISTINCT 关键字就显得特别有意义，它只获取唯一一次记录，而不是获取重复记录。

语法格式：

```postgresql

WHERE [condition]
```

```postgresql
-- 查询指定表中的所有列数据
SELECT * FROM <table_name>

-- 条件查询：组合查询，结合逻辑运算符
SELECT * FROM <table_name> WHERE <condition> [AND, OR, NOT] <condition>

SELECT * FROM <table_name> WHERE <column> LIKE '%xxx%';
-- 条件查询：指定列以 xxx 开头> LIKE 'xxx%';
-- 条件查询：指定列以 xxx 结尾
SELECT * FROM <table_name> WHERE <column> LIKE '%xxx';

SELECT * FROM <table_name> WHERE <column> IN (a, b);
SELECT * FROM <table_name> WHERE <column> NOT IN (a, b);
-- 条件查询：指定列的值在 [a, b] 区间范围以内的
SELECT * FROM students WHERE <column>  BETWEEN a AND b;
-- 查询数据并排序

-- 查询指定表中的指定列数据
SELECT column1, column2, ... FROM table_name [WHERE condition];
-- 分页查询
SELECT * FROM <table_name> LIMIT [no of rows] OFFSET [row num]
```

# 清空数据 TRUNCATE
