# 常用数据类型

## 数值类型

### 整数类型

- `INTEGER` / `INT`：4字节整数（-2147483648 到 +2147483647），**最常用**的整数类型
- `BIGINT`：8字节大整数（-9223372036854775808 到 +9223372036854775807），适合ID或大数值场景
- `SMALLINT`：2字节小整数（-32768 到 +32767），节省空间时使用
- `SERIAL` / `BIGSERIAL`：自增整数，**主键首选**（底层分别是INT和BIGINT）

### 小数类型

- `NUMERIC(p,s)` / `DECIMAL(p,s)`：精确小数（如金额），需指定精度(p)和小数位(s)
- `REAL`：4字节浮点数（约6位精度）
- `DOUBLE PRECISION`：8字节浮点数（约15位精度）

## 文本与字符串

- `TEXT`：**无长度限制**的变长文本，**推荐**替代VARCHAR
- `VARCHAR(n)`：变长字符串，需指定最大长度（如用户名、地址）
- `CHAR(n)`：定长字符串（不足补空格），适合固定编码（如ISO代码）

## 日期与时间

- `DATE`：纯日期（`'2025-06-19'`）
- `TIMESTAMP`：日期+时间（`'2025-06-19 14:30:00'`），**无时区**
- `TIMESTAMPTZ`：**带时区**的日期时间，**推荐**用于全球化系统
- `TIME`：纯时间（`'14:30:00'`）

## 其他核心类型

1. **布尔类型**
   - `BOOLEAN`：存储`TRUE`/`FALSE`/`NULL`（如状态标志）
2. **JSON支持**
   - `JSONB`：**二进制存储的JSON**，支持索引和高效查询，**现代应用首选**
   - `JSON`：文本格式JSON（无性能优化）
3. **特殊标识**
   - `UUID`：全局唯一标识符（如分布式ID）

# 使用建议

1. **优先选择**：
   - 整数用`INTEGER` 或 `BIGINT`
   - 文本用 `TEXT`
   - 时间用 `TIMESTAMPTZ`
   - JSON用 `JSONB`
2. **性能权衡**：
   - 精确计算选`NUMERIC`，非精确计算选`DOUBLE PRECISION`
   - 定长字段（如哈希值）用`CHAR`，变长字段用`TEXT`
