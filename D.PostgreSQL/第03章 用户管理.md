# 创建用户

```shell
# 终端交互式创建用户（推荐）
$ createuser --interactive 
# 或者直接指定用户名
$ createuser <user_name>
```

```postgresql
CREATE USER username WITH PASSWORD 'password'; -- 创建普通用户
CREATE USER root WITH SUPERUSER PASSWORD 'root_password'; -- 创建超级用户
```

# 查看用户

```postgresql
\du -- 查看用户列表
```

# 删除用户

```shell
$ dropuser <user_name>
```

```postgresql
DROP USER username;                      -- 删除用户
```

由于角色可以拥有数据库对象，并且可以拥有访问其他对象的权限，因此删除角色通常不仅仅是快速执行 `DROP USER` 的问题。必须首先删除该角色拥有的任何对象，或将其重新分配给其他所有者；并且必须撤销授予该角色的任何权限。

**如何安全删除PostgreSQL中已拥有数据库对象的角色？**

```postgresql
-- 将 doomed_role 拥有的所有数据库对象（表、视图、函数等）的所有权转移给 successor_role。
REASSIGN OWNED BY doomed_role TO successor_role;
-- 删除 doomed_role 在当前数据库中仍拥有的所有对象（如未转移的临时表），并回收该角色在其他对象上的权限
DROP OWNED BY doomed_role;
-- 在该集群的每个数据库中重复上述命令
-- 最终删除角色本身，确保该角色不再存在
DROP ROLE doomed_role;
```

# 修改密码

1）以超级管理员身份登录 PostgreSql

```shell
$ sudo -u <超级用户名> psql
$ sudo -u root psql
```

2）执行密码修改命令

```postgresql
ALTER USER 用户名 WITH PASSWORD '新密码';
```

3）退出

```postgresql
\q
```

4）验证

```shell
$ psql -U 用户名 -d 数据库名 -W
```

# 启用密码登录

1）查看安装目录

```shell
$ psql
```

```postgresql
SHOW data_directory;
/opt/homebrew/var/postgresql@17
```

2）进入目录里，找到并修改 `pg_hba.conf` 文件，将认证方法从`trust`改为`md5`或`scram-sha-256`（推荐），例如：

```python
# 本地登录强制密码验证
local   all             all                                     md5
# 远程登录强制密码验证
host    all             all             0.0.0.0/0               md5
```

- `md5`：使用MD5加密密码认证。
- `scram-sha-256`：更安全的SCRAM-SHA-256加密（PostgreSQL 10+支持）

3）重启PostgreSQL服务

```shell
brew services restart postgresql@17
```
