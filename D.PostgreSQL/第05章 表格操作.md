# 创建表格

PostgreSQL 使用 CREATE TABLE 语句来创建数据库表格。

## 语法结构

```postgresql
CREATE TABLE [IF NOT EXISTS] <table_name> (
   column1 datatype column_contraint,
   column2 datatype column_contraint,
   column3 datatype column_contraint,
   ......
   table_constraints
);
```

在这个语法中：

- 首先，在`CREATE TABLE`关键字后指定表的名称。
- 其次，创建一个已经存在的表会导致错误。`IF NOT EXISTS`选项允许您，仅当新表不存在时才创建新表。当您使用`IF NOT EXISTS`选项并且表已存在时，PostgreSQL 会发出通知而不是错误并跳过创建新表。
- 第三，指定以逗号分隔的表列列表。每列由列名、列存储的数据类型、数据长度和列约束组成。列约束指定存储在列中的数据必须遵循的规则。例如，非空约束强制列中的值不能为 NULL。列约束包括非空、唯一、主键、检查、外键约束。
- 最后，指定表约束，包括主键、外键和检查约束。

请注意，某些表约束可以定义为列约束，例如主键、外键、检查、唯一约束。

## 约束

@See https://www.runoob.com/postgresql/postgresql-constraints.html

PostgreSQL 约束用于规定表中的数据规则。

如果存在违反约束的数据行为，行为会被约束终止。

约束可以在创建表时规定（通过 CREATE TABLE 语句），或者在表创建之后规定（通过 ALTER TABLE 语句）。

约束确保了数据库中数据的准确性和可靠性。

约束可以是列级或表级。列级约束仅适用于列，表级约束被应用到整个表。

PostgreSQL 包含以下列约束：

- `NOT NULL`：确保列中的值不能为 `NULL`（空值）
  - 说明：在插入或更新数据时，被标记为 `NOT NULL` 的列必须提供非空值
- `UNIQUE`：确保列或列组合中的值唯一，不允许重复（但允许一个 `NULL` 值，除非显式指定 `NULLS NOT DISTINCT`）。
  - 说明：适用于需要避免重复数据的场景，如用户名或邮箱
- `PRIMARY KEY`：主键，唯一标识表中的每一行，是 `NOT NULL` 和 `UNIQUE` 的组合。
  - 说明：每个表只能有一个主键，通常用于外键引用的基础
- `FOREIGN KEY`：外键，维护表之间的引用完整性，确保外键列的值必须存在于被引用表的主键或唯一键中。
- `CHECK`：通过布尔表达式验证列或表级数据的有效性。
- `EXCLUSION`：确保指定列或表达式的值在范围内不重叠（如时间区间、空间数据）。
- `DEFAULT`：为列指定默认值，当插入数据未提供值时自动填充。

## 示例

### NOT NULL

默认情况下，列可以保存为 NULL 值。如果您不想某列有 NULL 值，那么需要在该列上定义此约束，指定在该列上不允许 NULL 值。

NULL 与没有数据是不一样的，它代表着未知的数据。

**实例**

下面实例创建了一张新表叫 COMPANY1，添加了 5 个字段，其中三个 ID，NAME，AGE 设置不接受空置：

```postgresql
CREATE TABLE COMPANY1 (
   id INT PRIMARY KEY NOT NULL,
   name TEXT NOT NULL,
   age INT NOT NULL,
   address CHAR(50)
   salary REAL
)
```

### UNIQUE

UNIQUE 约束可以设置列是唯一的，避免同一列出现重复值。

**实例**

下面实例创建了一张新表叫 COMPANY2，添加了 5 个字段，其中 AGE 设置为 UNIQUE，因此你不能添加两条有相同年龄的记录：

```postgresql
CREATE TABLE COMPANY2 (
   id INT PRIMARY KEY  NOT NULL,
   name TEXT NOT NULL,
   age INT NOT NULL UNIQUE,
   address CHAR(50),
   salary REAL DEFAULT 50000.00
);
```

### PRIMARY KEY

在设计数据库时，PRIMARY KEY 非常重要。

PRIMARY KEY 称为主键，是数据表中每一条记录的唯一标识。

设置 UNIQUE 的列可能有多个，但是一张表只有一列可以设置 PRIMARY KEY。

我们可以使用主键来引用表中的行，也可以通过把主键设置为其他表的外键，来创建表之间的关系。

主键是非空约束和唯一约束的组合。

一个表只能有一个主键，它可以由一个或多个字段组成，当多个字段作为主键，它们被称为复合键。

如果一个表在任何字段上定义了一个主键，那么在这些字段上不能有两个记录具有相同的值。

**实例**

下面我们创建 COMAPNY3 表，其中 ID 作为主键：

```postgresql
CREATE TABLE COMPANY3 (
   id INT PRIMARY KEY NOT NULL,
   name TEXT NOT NULL,
   age INT NOT NULL,
   address CHAR(50),
   salary REAL
);
```

### FOREIGN KEY

FOREIGN KEY 即外键约束，指定列(或一组列)中的值必须匹配另一个表的某一行中出现的值。

通常一个表中的 FOREIGN KEY 指向另一个表中的 UNIQUE KEY(唯一约束的键)，即维护了两个相关表之间的引用完整性。

**实例**

下面实例创建了一张 COMPANY4 表，并添加了5个字段：

```postgresql
CREATE TABLE COMPANY4 (
   id INT PRIMARY KEY NOT NULL,
   name TEXT NOT NULL,
   age INT NOT NULL,
   address CHAR(50),
   salary REAL
);
```

下面实例创建一张 DEPARTMENT1 表，并添加 3 个字段，EMP_ID 就是外键，参照 COMPANY4 的 ID：

```postgresql
CREATE TABLE DEPARTMENT1(
   id INT PRIMARY KEY NOT NULL,
   dept CHAR(50) NOT NULL,
   emp_id INT references COMPANY4(ID)
);
```

### CHECK 约束

CHECK 约束保证列中的所有值满足某一条件，即对输入一条记录要进行检查。如果条件值为 false，则记录违反了约束，且不能输入到表。

**实例**

例如，下面实例建一个新的表 COMPANY5，增加了五列。在这里，我们为 SALARY 列添加 CHECK，所有工资不能为零：

```postgresql
CREATE TABLE COMPANY5 (
   id INT PRIMARY KEY NOT NULL,
   name TEXT NOT NULL,
   age INT NOT NULL,
   address CHAR(50),
   salary REAL CHECK(salary > 0)
);
```

# 删除表

要从数据库中删除表，请使用`DROP TABLE`语句，如下：

```postgresql
DROP TABLE [IF EXISTS] table_name [CASCADE | RESTRICT];
```

在这个语法中：

- 首先，在 `DROP TABLE` 关键字后指定要删除的表的名称。
- 其次，使用 `IF EXISTS` 选项时，仅当表存在才删除该表。

如果删除不存在的表，PostgreSQL 会发出错误。为了避免这种情况，您可以使用 `IF EXISTS` 选项。

如果要删除的表已在其他对象（例如[视图](https://www.rockdata.net/zh-cn/tutorial/ddl-views/)、[触发器](https://www.rockdata.net/zh-cn/tutorial/ddl-triggers/)、函数和[存储过程](https://www.rockdata.net/zh-cn/tutorial/plpgsql-overview/)）中使用，则`DROP TABLE` 无法删除该表。在这种情况下，您有两种选择：

- `CASCADE` 选项允许您删除表及其依赖对象。
- 如果有任何对象依赖于表，`RESTRICT` 选项将拒绝删除。如果您未在 `DROP TABLE` 语句中明确指定，则 `RESTRICT` 选项为默认行为。

要一次删除多个表，您可以在`DROP TABLE`关键字后面放置一个以逗号分隔的表列表：

```postgresql
DROP TABLE [IF EXISTS] 
   table_name_1,
   table_name_2,
   ...
[CASCADE | RESTRICT];
```

请注意，您需要具有超级用户、模式所有者或表所有者的角色，才能删除表。

# 修改表

@See https://www.runoob.com/postgresql/postgresql-alter-table.html

在 PostgreSQL 中，**ALTER TABLE** 命令用于添加，修改，删除一张已经存在表的列。

另外你也可以用 **ALTER TABLE** 命令添加和删除约束。

语法格式：

```postgresql
-- 01. 修改表名
ALTER TABLE [IF EXISTS] 原表名 RENAME TO 新表名;
-- 02. 修改字列名
ALTER TABLE [IF EXISTS] 表名 RENAME COLUMN 原字段名 TO 新字段名;
-- 03. 添加列
ALTER TABLE table_name ADD column_name datatype;
-- 04. 删除列
ALTER TABLE table_name DROP COLUMN column_name;
-- 05. 修改列的数据类型
ALTER TABLE table_name ALTER COLUMN column_name TYPE datatype;
-- 06. 给表中某列添加 NOT NULL 约束
ALTER TABLE table_name ALTER column_name datatype NOT NULL;
-- 07. 给表中某列添加 UNIQUE 约束
ALTER TABLE table_name
ADD CONSTRAINT MyUniqueConstraint UNIQUE(column1, column2...);
-- 08. 给表中添加 CHECK 约束
ALTER TABLE table_name
ADD CONSTRAINT MyUniqueConstraint CHECK (CONDITION);
-- 09. 给表添加主键
ALTER TABLE table_name
ADD CONSTRAINT MyPrimaryKey PRIMARY KEY (column1, column2...);
-- 10. 删除约束
ALTER TABLE table_name
DROP CONSTRAINT MyUniqueConstraint;
```
